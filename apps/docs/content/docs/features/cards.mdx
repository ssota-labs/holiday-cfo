## 개요

청구주기는 카드 부채 계정에 붙는 날짜 규칙이다. 없으면 원장은 빚진 금액은 알아도 현금이 언제 빠지는지는 모른다. `cashflow`의 카드 청구 투영은 이 규칙에 의존한다.

규칙은 세 값으로 구성된다.

| 필드 | CLI 플래그 | 의미 |
|---|---|---|
| `cycleCloseDay` | `--close-day` | 사용기간이 닫히는 날(포함). 이 날 구매는 해당 청구서 |
| `paymentMonthOffset` | `--payment-month-offset` | 마감월에서 결제월까지 개월 수. 보통 0 또는 1 |
| `paymentDay` | `--payment-day` | 결제 일. `-1`이면 해당 월 말일 |

## 동작

`billingDatesFor(purchaseDate, rule)`가 구매일에 대해 `closeDate`와 `paymentDate`를 계산한다.

1. 구매일이 당월 `cycleCloseDay`(말일 clamp) 이하면 당월 마감, 초과면 익월 마감.
2. 결제월 = 마감월 + `paymentMonthOffset`.
3. 결제일은 `paymentDay`를 해당 월 일수에 clamp. `-1`은 말일.
4. `paymentDate < closeDate`이면 규칙을 거부한다. offset을 늘려야 한다.

`--close-day 31`은 "말일 마감"이다. 2월에는 28(또는 29)일로 clamp된다. `--payment-day -1`도 동일하게 말일이다.

`upcomingBills`는 커서를 오늘 이전이 아니라 **더 과거**에서 시작한다. 이미 마감됐지만 아직 결제되지 않은 청구서가 가장 임박한 유출이다. [POLICY-008](/docs/domain/policy#policy-008)

등록·목록 출력은 추상 규칙뿐 아니라 "오늘 구매하면 현금이 빠지는 날" 예시를 함께 보여 준다.

## 제약

- `cycleCloseDay` ∈ [1, 31]. `paymentDay` ∈ [1, 31] 또는 `-1`. `paymentMonthOffset` ∈ [0, 3].
- 카드 계정과 funding 계정은 이미 `account add`로 존재해야 한다.
- 일반 청구액 합산은 카드 계정 posting만 본다. 할부 잔액 계정(`*:Installment`)은 제외된다.
- 결제 거래(상대 leg가 funding)는 청구액에서 제외한다. [POLICY-004](/docs/domain/policy#policy-004)
