홈택스·종합소득세 신고서에 적힌 **소득금액·과세표준·납부할세액** 같은 숫자를
장부(로컬 DB)에 남겨 둔다. 통장에서 나간 세금(전표)과는 **다른 기록**이다.

## 신고 한 건에 들어가는 정보

### 신고 표지

| 항목 | 의미 |
|---|---|
| 신고서 종류 | 1차는 종합소득세 |
| 귀속연도 | 몇 년도 소득에 대한 신고인지 |
| 신고일 | 제출한 날짜 |
| 신고 차수 | 최초 신고 / 1차 수정 / 2차 수정… |
| 메모·원본 | 선택. PDF 경로나 짧은 메모 |

### 세액표 숫자

신고서 **세액계산** 칸을 그대로 옮긴다. 세액을 다시 계산하지 않는다.

대표 칸: 소득금액, 소득공제, 과세표준, 세율, 산출세액, 세액감면·공제, 결정세액,
가산세, 기납부세액, 납부할세액.  
종합소득 열에는 최소한 **소득금액·과세표준·납부할세액**이 있어야 한다.

칸·예시 표는 [prd-tax-return-sor](/docs/planning/prds/prd-tax-return-sor)에 있다.
입력할 때 쓰는 JSON 키는 [`tax return add`](/docs/spec/cli/tax/return-add)를 본다.

## 수락 기준

1. 표지와 세액표 숫자를 한 번에 저장한다 ([POLICY-018](/docs/domain/policy#POLICY-018))
2. 같은 종류·귀속연도·차수를 다시 넣으면 거절한다
3. 수정신고는 새 차수를 추가하고, 이전 차수는 “대체됨”으로 남긴다
4. 세액·공제 공식을 추정하지 않는다 — 신고서에 찍힌 숫자만
5. 신고서 저장이 전표를 만들지 않는다 (납부는 `txn` + `Expenses:Tax:*`)

## 계정과의 관계

| 질문 | 보는 곳 |
|---|---|
| 통장에서 나간 세금 / 비용으로 잡은 세금 | `Expenses:Tax:*`, `Assets:Receivable:Tax`, `Liabilities:Payable:Tax` |
| 신고서의 소득금액·과세표준·결정세액 | `holiday tax return show` |

둘은 다른 질문이다. 신고서 기록이 생겨도 전표 규칙은 그대로다.
