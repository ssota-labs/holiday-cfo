장부 화면(dash)은 **사진**이다. 장부를 브라우저가 직접 열지 않고, CLI가 한
시점에 구운 숫자 스냅샷을 읽는다.

## 역할 분리

| 누가 | 무엇을 |
|---|---|
| CLI (`dash data`) | 잔액·현금 여유·장부 건강 스냅샷 |
| 사용자/에이전트 | 페이지 배치·설명 문장 |
| 닫힌 칸 | 스냅샷만 읽어 표·런웨이·분류 대기를 그림 |

로컬 미리보기에서만 분류 대기를 클릭으로 처리한다. 배포본은 읽기 전용이다
([ADR-008](/docs/development/adr/adr-008)).

## 작성 모델 (계획)

배치 정본을 JSON 한 장에서 **MDX 페이지 트리**로 옮긴다
([prd-dash-mdx-shell](/docs/planning/prds/prd-dash-mdx-shell),
[ADR-012](/docs/development/adr/adr-012)). 문서 사이트와 같은 MDX 등록·타입
계약에 대시보드 칸을 얹고, DocsLayout/Next 문서 런타임은 이식하지 않는다.

## 하지 않는 것

- 라이브 잔액 구독
- 화면에서 세액·할부 공식 재계산
- 페이지에 금액을 다시 적어 “현재 잔액”처럼 보이게 하기
