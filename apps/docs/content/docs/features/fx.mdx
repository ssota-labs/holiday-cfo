## 개요

환율은 기표 시점의 weight를 **유도**하거나, 마감 시 외화 화폐성 계정 장부가액을 **재평가**할 때만 쓴다. 이미 저장된 `weight_minor`를 과거로 되돌려 다시 쓰지 않는다.

저장은 소수 문자열, 연산은 scaled bigint다. float를 쓰지 않는다.

의미: `1 <base> = <rate> <quote>`.

## 동작

### resolve 순서

`resolveRate`는 아래 순서로만 찾는다. 한 단계가 실패하면 다음으로 내려간다. 없으면 throw한다. **1.0으로 대체하지 않는다.**

1. **exact** — 요청일(`asOf`)에 해당 페어가 있음
2. **stale** — 같은 페어의 최근 환율이 `maxStalenessDays` 이내
3. **inverse** — 반대 페어의 역수(staleness 적용)
4. **triangulated** — 기능통화를 경유 (`base → functional → quote`)
5. **throw** — 없음. `holiday fx add` 또는 leg에 `@@`로 총액을 직접 기입

`fx show`는 실제로 어떤 kind가 선택됐는지와 explanation을 출력한다.

### weight와 @@

외화 leg의 weight 출처:

| 방식 | 의미 |
|---|---|
| `@@ TOTAL` | 관측 총액(기능통화). `weight_source: 'actual'`. 명세서·영수증에 찍힌 KRW |
| 환율 유도 | `resolveRate` + `convert`. `weight_source: 'rate'`. 추정 |

`@@`는 **총액**이다. 단위당 `@` 환율 문법은 쓰지 않는다. 환율을 되곱하면 정수에 안 떨어지고, 허용오차를 부르는 경로가 된다.

환율을 나중에 수정·추가해도 **이미 posted된 weight는 변하지 않는다.** 이후 기표의 유도와 마감 재평가에만 반영된다.

## 제약

- 환율 없음은 질문이므로 기본값 1.0을 넣지 않는다.
- posted weight는 환율 테이블 변경으로 재계산되지 않는다.
- 재평가는 `close`가 화폐성 자산·부채에만 수행한다. 비용·수익 계정은 거래일 측정으로 둔다.
- 동일 통화 페어는 rate = 1(스케일) exact로 처리한다.
