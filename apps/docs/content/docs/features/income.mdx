두 층이다.

1. **정기수입** (`income add`) — cashflow용 **실수령** 예측. 미도래분만.
2. **수입 정산** (`income source` / `settle`) — 업체별 regime으로 법정 공제를 **강제 계산**.

## 수락 기준

1. today 이하 발생은 투영하지 않는다 ([POLICY-017](/docs/domain/policy#POLICY-017))
2. 입금 계정이 `--cash`가 아니면 런웨이에서 빠지고 gap으로 알린다
3. 사업소득 원천은 3.3% ([POLICY-018](/docs/domain/policy#POLICY-018))
4. 급여는 갑근세 입력 필수 ([POLICY-019](/docs/domain/policy#POLICY-019))
5. 저장된 정산은 법정 계산과 일치 ([POLICY-020](/docs/domain/policy#POLICY-020))

## Regime

| regime | 용도 |
|---|---|
| `business_withholding` | 프리랜서·사업소득 원천징수 3.3% |
| `business_vat` | 세금계산서 부가세 10% (공급가 기준) |
| `salary` | 근로소득 — 4대보험 + 갑근세 |
| `allowance` | 공제 없는 수당 |

요율·개정: 스킬 `references/workflows/kr-income.md`와 `packages/core/src/domain/kr-income.ts`.
