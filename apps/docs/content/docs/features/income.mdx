수입은 두 층이다.

1. **정기수입** (`income add`) — 앞으로 들어올 **실수령**만 현금흐름에 올린다.
   오늘·과거분은 이미 기표했다고 본다.
2. **수입 정산** (`income source` / `settle`) — 업체별 방식으로 원천징수·부가세·
   4대보험을 **법령대로** 맞춘다.

## 정기수입에 들어가는 정보

| 항목 | 의미 |
|---|---|
| 실수령액 | 통장에 찍힐 금액 (예측) |
| 입금 계정 | 보통 `--cash` 통장. 아니면 런웨이에서 빠짐 |
| 주기 | 매달·격주 등 |

## 정산(regime)

| regime | 용도 |
|---|---|
| `business_withholding` | 프리랜서·사업소득 원천징수 3.3% |
| `business_vat` | 세금계산서 부가세 10% (공급가 기준) |
| `salary` | 근로소득 — 4대보험 + 갑근세 |
| `allowance` | 공제 없는 수당 |

요율·개정: 스킬 `references/workflows/kr-income.md`와
`packages/core/src/domain/kr-income.ts`.

## 수락 기준

1. 오늘 이하 발생분은 투영하지 않는다 ([POLICY-017](/docs/domain/policy#POLICY-017))
2. 입금 계정이 `--cash`가 아니면 런웨이에서 빼고 gap으로 알린다
3. 사업소득 원천은 3.3%로 맞춘다 ([POLICY-018](/docs/domain/policy#POLICY-018))
4. 급여 정산은 갑근세 입력 없이 끝낼 수 없다 ([POLICY-019](/docs/domain/policy#POLICY-019))
5. 저장된 정산은 법정 계산과 일치한다 ([POLICY-020](/docs/domain/policy#POLICY-020))
