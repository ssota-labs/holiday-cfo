장부에 어떤 계좌와 부채가 있는지를 **마크다운 한 장**에 적어 둔다. 잔액·전망·마감
숫자는 넣지 않는다. 그 숫자는 잔액 조회·현금흐름·대시보드에서 본다.

## 언제 보나

1. **사람이** 에디터에서 구조를 훑을 때
2. **에이전트가** 세션 초반에 방향을 잡을 때 — 파일이 없으면 목록 조회로 넘어간다
3. **계정·카드·대출·할부를 바꾼 뒤** 다시 써서 목록을 맞출 때
4. **세션이 열릴 때** — 훅이 켜진 호스트에서는 파일이 없으면 만들어 둔다. 있으면
   그대로 둔다

`holiday status`를 실행하면 프로젝트 루트의 `status.md`를 원장 기준으로 통째로
다시 쓴다. `holiday status --json`은 파일 경로와 갱신 시각을 반환한다.

세션 훅(Claude Code·Codex 플러그인, 새 장부의 Cursor 템플릿)은 파일이 **없을
때만** 같은 다시 쓰기로 만든다. 이미 있는 Cursor `hooks.json`은 init이 덮지
않는다 — 수동 추가는 [SPEC-ledger-status](/docs/agent/ledger-status)의
「기존 Cursor 장부」절을 본다.

## 감각

| 구분 | 이 파일 | 장부 DB |
|---|---|---|
| 역할 | 읽기용 나열 | 진실 |
| 손으로 고침 | 다시 쓰면 덮임 | 계정·스케줄 명령으로만 변경 |
| 잔액 | 없음 (1차) | 잔액 조회 |

## 수락 기준

1. `status.md`에 계좌·부채 이름과 갱신 시각이 있다
2. 다시 쓰기 후 장부에 있는 계좌·부채가 목록에 나타난다
3. 파일만 고쳐서는 장부가 바뀌지 않는다
4. 에이전트 안내에 현황 파일을 먼저 확인한다는 내용이 있다
5. 세션 훅이 켜진 호스트에서 파일이 없으면 생기고, 있으면 유지된다
