## 개요

정기지출(월세·구독·보험·통신)은 스케줄이다. 원장 계정이 아니다. 가격 변경·해지 시 과거를 오염시키지 않기 위해 예측으로만 둔다. 실제로 빠져나가면 일반 `txn`으로 기록한다.

funding 계정 종류가 현금 이탈일을 결정한다.

| funding | 발생일 | 현금 이탈 |
|---|---|---|
| 자산(통장) | 카덴스 발생일 | 같은 날 |
| 부채(카드, 청구주기 있음) | 카드에 청구만 생김 | 해당 청구주기 결제일 |

같은 금액·같은 카덴스라도 통장 월세와 카드 구독은 수주 어긋난다. [POLICY-010](/docs/domain/policy#policy-010)

## 동작

카덴스:

- `monthly` + `--day` (기본 1, `-1` = 말일)
- `--yearly <month>`이면 `yearly` (해당 월 + day)

활성 구간은 `--from` ~ `--to`(생략 시 무기한). `isActiveOn`이 발생일마다 검사한다.

`cashflow`의 `projectRecurring`은 **오늘 초과** 발생분만 투영한다. 오늘 이하 발생은 원장 posting으로 이미 잡혔다고 본다. 카드 funding이면 일반 청구 투영과 이중 계상하지 않기 위함이다. [POLICY-005](/docs/domain/policy#policy-005)

카드 funding이면 출력에 발생일을 붙인다. 예: `넷플릭스 (2026-08-17 결제분)`.

`recurring add`는 funding이 카드면 "오늘 청구 시 결제일" 예시를, 통장이면 "당일 인출"을 안내한다.

```bash

```

## 제약

- 발생일 ≤ today인 항목은 `cashflow`에 넣지 않는다. [POLICY-005](/docs/domain/policy#policy-005)
- 카드 funding은 해당 계정에 청구주기가 있어야 현금 이탈일을 계산한다. 없으면 카드로 취급되지 않고 funding 당일로 본다(카드 규칙 맵에 없음).
- dayOfMonth ∈ [1, 31] 또는 `-1`. yearly month ∈ [1, 12].
- 스케줄은 예측이다. 실제 인출·카드 청구는 별도 거래로 기록한다.
