장부의 기본 칸이다. `Assets:Bank:KB`처럼 코드로 계정을 만들고, 균형 잡힌 전표를
넣으면 잔액이 쌓인다. 계정 종류(자산·부채·자본·수익·비용)는 코드 앞부분으로 정한다.

## 계정 한 건에 들어가는 정보

| 항목 | 의미 |
|---|---|
| 코드 | `Assets:…` / `Liabilities:…` 등. 앞부분이 종류 |
| 현금 표시 | 통장처럼 런웨이 시작 잔액에 넣을지 (`--cash`) |
| placeholder | 자리만 잡은 계정. 여기에 전표를 못 넣음 |

잔액은 전표의 **weight**(장부가액) 합이다. 현금 여부는 코드 prefix가 아니라
플래그로 본다 ([ADR-004](/docs/development/adr#ADR-004)).

## 수락 기준

1. `--cash`로 표시한 계정이 현금흐름 시작 잔액에 들어간다
2. placeholder 계정에는 전표를 넣을 수 없다
3. 차변·대변이 안 맞는 전표는 거절한다 ([POLICY-001](/docs/domain/policy#POLICY-001))
