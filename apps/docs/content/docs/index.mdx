`holiday`는 1인용 복식부기 원장 CLI다. 로컬 SQLite(`.holiday/ledger.db`)에 거래·스케줄·환율·마감을 기록하고, 현금흐름과 감사 체인을 검증한다.

런타임은 **Node ≥ 24**다. CLI 저장소는 SQLite뿐이다. 배포는 `npx @holiday-cfo/cli` (ADR-007).

## 이 문서를 읽는 순서

1. [Vision](/docs/planning/vision) · [PRDs](/docs/planning/prds) — 왜, 이번 주기에 무엇을

6. [설계](/docs/design/system) · [에이전트](/docs/agent/skill)

## PRD vs Features

| | PRD | Features |
|---|---|---|
| 시간축 | 기간/이니셔티브에 묶여 **추가** | 상시 카탈로그 |
| 질문 | 이번에 무엇을 만들까? | 제품이 지금 무엇을 할 수 있나? |
| 위치 | [`planning/prds`](/docs/planning/prds) | [`features`](/docs/features/accounts) |

## 빠른 시작

```bash
holiday init --currency KRW
holiday account add Assets:Bank:KB:Checking --commodity KRW --cash
holiday account add Liabilities:Card:Shinhan --commodity KRW
holiday card add Liabilities:Card:Shinhan --funding Assets:Bank:KB:Checking \
  --close-day 14 --payment-day 1
holiday txn add --payee "예" --leg "Expenses:Food 12000 KRW" \
  --leg "Assets:Bank:KB:Checking -12000 KRW"
holiday cashflow --until 2026-09-30
holiday verify
```

## 제약 (코드 기준)

- OCR·이미지 파싱 없음 (`ingest`는 이미 파싱된 JSON)
- 환율 자동 수집 API 없음
- 라이브 대시보드 없음 (`dash`는 스냅샷 굽기)
- CLI는 SQLite만 (`@holiday-cfo/store-postgres`는 라이브러리/테스트용)
