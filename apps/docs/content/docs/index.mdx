`holiday`는 에이전트가 캡쳐를 읽고 CLI를 호출하면 장부가 서는 도구다. CLI가
복식부기 불변식·중복제거·환율·스케줄을 책임지고, 에이전트는 읽고 옮기는 일만 한다.

## 이 문서가 특이한 이유

**이 프로젝트는 정책이 곧 제품이다.** "허용오차가 없다", "할부는 별도 계정이어야
한다", "발생하지 않은 것만 투영한다" — 이건 구현 세부가 아니라 숫자를 믿을 수 있는
이유 전부다.

그래서 [정책](/docs/policy)의 모든 규칙은 **그걸 강제하는 테스트에 링크된다.**
규칙이 조용히 거짓이 될 수 없다 — 어기면 CI가 빨개진다.

## 어디서 무엇을 찾나

문서를 형태(설계/참조)가 아니라 **누가 묻느냐**로 나눴다.

## 지금 있는 것

원장은 선다. 복식부기·다통화·감사 체인이 동작하고, 네 스케줄이 현금흐름 하나로 모이며,
캡쳐는 드래프트로 들어와 사람 승인을 거친다. 환율·잔고 단언·월 마감(FX 재평가 포함)도
동작한다.

**아직 없는 것:** 환율 자동 수집(직접 넣어야 한다), Supabase 어댑터의 store 본체(스키마와
트리거는 있다), 대시보드.

## 버전은 스펙마다 따로다

사이트 전체 버전은 없다. **거짓말이기 때문이다** — 데이터 모델은 마이그레이션 4개를
거쳤고, CLI는 0.1.0이며, 도메인 정책은 버전이 아예 없다(지금 참이거나 틀렸거나 둘 중
하나다). 하나의 번호를 셋에 찍으면 같이 움직인다고 말하는 셈이다.

각 스펙 페이지 맨 위에 자기 버전과 이력이 있고, **각 항목이 자기를 증명하는 산출물을
가리킨다** — 마이그레이션 폴더, package.json. 근거를 못 대는 changelog는 한 달이면
표류한다.

## 에이전트로 읽기

모든 페이지에 `.md` 쌍둥이가 있다. `/docs/policy`는 React를 렌더하고,
[`/docs/policy.md`](/docs/policy.md)는 원문 마크다운을 준다.

이 도구를 운전하는 게 에이전트라서, 이건 부가 기능이 아니다 —
`plugins/claude-code/skills/holiday-cfo/`의 스킬과 이 사이트를 같은 것이 읽는다.
