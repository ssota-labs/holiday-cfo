`holiday`는 1인용 복식부기 원장 CLI다. 거래·계정·카드 청구주기·할부·정기지출·대출을
기록하고, 현금흐름 예측과 감사 체인 검증을 제공한다.

런타임은 **Node ≥ 24**다(`node:sqlite`, `--experimental-strip-types`). CLI가 쓰는
저장소는 작업 디렉터리의 로컬 파일 `.holiday/ledger.db`(SQLite)뿐이다.

## 빠른 시작

```bash
holiday init --currency KRW
holiday account add Assets:Bank:KB:Checking --commodity KRW --cash
holiday account add Liabilities:Card:Shinhan --commodity KRW
holiday card add Liabilities:Card:Shinhan --funding Assets:Bank:KB:Checking \
  --close-day 14 --payment-day 1
holiday txn add --payee "예" --leg "Expenses:Food 12000 KRW" \
  --leg "Assets:Bank:KB:Checking -12000 KRW"
holiday cashflow --until 2026-09-30
holiday verify
```

일반적인 순서는 `init` → `account` → `card` → `txn` → `cashflow` → `verify`다.
기능통화는 `init --currency`로 정한다. 바꾸려면 새 원장을 연다.

## 현재 기능

| 영역 | 명령·동작 |
| --- | --- |
| 데이터 수집·검토 | `ingest submit`으로 draft 적재 → `review`로 posted/rejected |
| 환율 | `fx add` / `list` / `show`. 수동 입력만 |
| 잔고 단언·마감 | `assert`, `close` (게이트 검사 → FX 재평가 → 스냅샷 → `period.status=closed`) |
| 스케줄 | 카드 청구주기, 할부, 정기지출, 대출 |
| 현금흐름 | `cashflow` — 현금 계정에서 스케줄 지출을 투영 |
| 감사 | `verify` — 불변식·해시 체인 |

에이전트/사람은 캡쳐를 파싱해 CLI에 넘긴다. CLI는 OCR을 수행하지 않는다.

## 제약

- OCR·이미지 파싱 없음 (`ingest`는 이미 파싱된 JSON을 받는다)
- 환율 자동 수집 API 없음 (`fx add`로 직접 기록)
- 대시보드·웹 UI 없음 (제품은 CLI)
- CLI 저장소는 SQLite만 (`init`이 `store: 'sqlite'` 고정). `@holiday/store-postgres`는
  라이브러리/테스트용이며 CLI에서 선택하지 않는다

## 버전

사이트 전역 단일 버전 번호는 없다. 데이터 모델은 마이그레이션으로, CLI는
`package.json` 버전으로, 정책은 테스트 링크로 각각 추적한다. 스펙 페이지의
`<SpecVersion>`이 해당 영역의 근거 산출물을 가리킨다.

## 에이전트용 원문

각 문서 경로에 `.md` 쌍이 있다. 예: [`/docs/domain/policy.md`](/docs/domain/policy.md).
`plugin/skills/holiday-cfo/` 스킬과 이 문서를 같은 독자가 읽는다.
