현재 기준 소스는 `packages/store-sqlite/src/schema.drizzle.ts`, 최신 마이그레이션은
`20260719045457_loose_sunspot`, 생성 스냅샷은
`packages/store-sqlite/migrations/20260719045457_loose_sunspot/snapshot.json`이다.
아래 표와 그래프는 이 스냅샷에서 생성된 manifest를 그대로 읽는다.

현재 Drizzle 테이블은 **25개**다. `tax_return`과 `tax_return_line`은 계획 단계이며 현재
스키마, 그래프, 스냅샷에 포함되지 않는다. SQLite 드라이버가 관리하는
`__holiday_migrations`는 이름과 sha256을 기록하는 마이그레이션 원장으로, 25개 도메인
Drizzle 테이블과 별도로 취급한다 ([POLICY-016](/docs/domain/policy#POLICY-016)).

## 사실 / 예측 / 정산 / 파생

| 종류 | 예 | 저장 |
|---|---|---|
| 사실 (fact) | `txn`, `posting`, `fx_rate`, `balance_assertion` | 저널과 관측 테이블 |
| 예측 (forecast) | `card`, `installment`, `recurring`, `recurring_income`, `loan`과 각 스케줄 행 | 저널 밖 |
| 정산 (settlement) | `income_source`, `income_settlement`, `income_settlement_line` | 저널 밖, 선택적 `txn` 링크 |
| 파생 (derived) | balance, cashflow | 저장하지 않고 조회 시 집계 |

예측과 정산 행은 현금흐름을 설명하지만 전기가 아니다. 실제로 일어난 일만 `txn`과
`posting`으로 확정한다.

## 불변식 링

1. Domain — `Txn.create`가 정확한 `SUM(weight_minor) = 0`을 검증한다
   ([POLICY-001](/docs/domain/policy#POLICY-001), [POLICY-002](/docs/domain/policy#POLICY-002)).
2. Port — `ValidatedTxn`만 append할 수 있다.
3. Storage — PK, FK, UNIQUE, CHECK와 트리거가 저장 시점 불변식을 막는다.
4. Audit — append-only `audit_log`가 content hash chain을 잇는다.

## 버전 메모

- `migration-8`은 SQLite 마이그레이션 폴더 순서다.
- `book.schema_version`은 장부 포맷 계약 값이며 마이그레이션 개수와 같은 숫자가 아니다.
- Postgres 엔진은 같은 논리 모델을 squash한 별도 마이그레이션 이력을 가진다.
- 적용된 마이그레이션의 sha256은 바뀌지 않는다. 스키마 변경은 새 마이그레이션을 append한다.

경계와 데이터 흐름은 [시스템 모델](/docs/spec/system-model)을 본다.
