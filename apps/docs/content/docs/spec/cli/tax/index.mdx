## Forms / periods

| `--form` | 의미 | `--period` |
|---|---|---|
| `kr_global_income` (기본) | 종합소득세 | 생략 시 `annual` (다른 값 거부) |
| `kr_vat` | 부가가치세 | **필수** — `H1_provisional` · `H1_final` · `H2_provisional` · `H2_final` |

### `--data-file` — 종합소득세 (`kr_global_income`)

```json
{
  "commodity": "KRW",
  "columns": {
    "global_income": {
      "income_amount": "22144107",
      "income_deduction": "2350580",
      "tax_base": "19793527",
      "tax_rate": "15.0",
      "calculated_tax": "1709029",
      "tax_reduction": "0",
      "tax_credit": "70000",
      "determined_tax_global": "1639029",
      "determined_tax_separate": "0",
      "determined_tax_total": "1639029",
      "additional_tax": "107570",
      "additional_tax_payable": "0",
      "total_tax": "1746599",
      "already_paid": "563323",
      "net_tax": "1183276",
      "payable_within_deadline": "1183276"
    },
    "rural_special": {
      "tax_base": "0",
      "tax_rate": "0.0",
      "calculated_tax": "0",
      "determined_tax_total": "0",
      "net_tax": "0",
      "payable_within_deadline": "0"
    }
  }
}
```

- 금액은 **소수 문자열**(기능통화 minor로 파싱). `number` 금지.
- `tax_rate`는 퍼센트 문자열(`"15.0"`). `RATE_SCALE` 스케일드 bigint로 저장.
- `global_income`에는 `income_amount`·`tax_base`·`payable_within_deadline`이 **필수**.

### `--data-file` — 부가가치세 (`kr_vat`)

```json
{
  "commodity": "KRW",
  "columns": {
    "vat": {
      "taxable_supply": "48000000",
      "output_tax": "4800000",
      "input_tax": "1200000",
      "payable_tax": "3600000",
      "refundable_tax": "0",
      "provisional_paid": "0",
      "additional_tax": "0"
    }
  }
}
```

- `vat`에는 `taxable_supply`·`output_tax`·`input_tax`가 **필수**.
- `payable_tax`와 `refundable_tax` 중 **최소 하나** 필수.
- CLI는 라인 간 산술 관계를 검증·수정하지 않는다 (관측 SoR).

예:

```bash
holiday tax return add --form kr_vat --year 2025 --period H1_final \
  --filed-on 2025-07-25 --data-file vat-h1.json --json
```
