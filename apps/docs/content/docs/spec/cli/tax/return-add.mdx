## `--data-file` 스키마

```json
{
  "commodity": "KRW",
  "columns": {
    "global_income": {
      "income_amount": "22144107",
      "income_deduction": "2350580",
      "tax_base": "19793527",
      "tax_rate": "15.0",
      "calculated_tax": "1709029",
      "tax_reduction": "0",
      "tax_credit": "70000",
      "determined_tax_global": "1639029",
      "determined_tax_separate": "0",
      "determined_tax_total": "1639029",
      "additional_tax": "107570",
      "additional_tax_payable": "0",
      "total_tax": "1746599",
      "already_paid": "563323",
      "net_tax": "1183276",
      "payable_within_deadline": "1183276"
    },
    "rural_special": {
      "tax_base": "0",
      "tax_rate": "0.0",
      "calculated_tax": "0",
      "determined_tax_total": "0",
      "net_tax": "0",
      "payable_within_deadline": "0"
    }
  }
}
```

- 금액은 **소수 문자열**(기능통화 minor로 파싱). `number` 금지.
- `tax_rate`는 퍼센트 문자열(`"15.0"`). 스케일드 bigint로 저장.
- 키가 빠진 라인은 저장하지 않는다. `kr_global_income`의 `global_income` 컬럼에는
  `income_amount`·`tax_base`·`payable_within_deadline`이 **필수**.
- CLI는 라인 간 산술 관계를 검증·수정하지 않는다 (관측 SoR).
