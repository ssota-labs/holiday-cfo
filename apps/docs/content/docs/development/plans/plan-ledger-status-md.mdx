## 목표

[prd-ledger-status-md](/docs/planning/prds/prd-ledger-status-md)대로 장부 프로젝트
루트에 `status.md`를 두고, 계좌·부채 이름만 나열한다. SoR는 DB다.

계약: [SPEC-ledger-status](/docs/agent/ledger-status).

## 영향 범위

| 영역 | 변경 |
|---|---|
| `packages/core` | 현황 스냅샷 조립(순수) — 자산 잎·카드·대출·할부·기타 부채 |
| `packages/cli` | 다시 쓰기 명령 + 마크다운 렌더 + (선택) 구조 변경 성공 시 자동 다시 쓰기 |
| `apps/docs` | 기능·에이전트 스펙 정합, CLI 스펙 페이지 추가 |
| 플러그인 | SKILL/장부 템플릿: 구조 파악 시 `status.md` 우선, 폴백 목록 명령 |

`store-*` 마이그레이션 없음.

## 구현 순서

1. core: 현황 DTO + 조립 함수 (placeholder 제외, 현금 플래그, 스케줄 라벨)
2. core 테스트로 SPEC 수락 기준 1–3 고정 (금액 칸 없음 포함)
3. CLI: 다시 쓰기 명령 — cwd의 `status.md` 덮어쓰기, `note()`·`--json`
4. (권장) `account`/`card`/`loan`/`installment` 변경 성공 경로에서 같은 다시 쓰기 호출
5. CLI 스펙 페이지 + 기능/에이전트 문서 링크
6. 플러그인 SKILL + `templates/ledger/AGENTS.md`: 구조 파악 → `status.md` → 폴백
7. `pnpm --filter @holiday-cfo/core test` · 관련 CLI 테스트 · `check:skill`

명령 이름은 구현 PR에서 CLI 스펙에 고정한다. 기획 본문은 다시 쓰기 명령으로만
부른다.

## 검증 계획

### 자동 테스트

- 자산 잎 + 현금 플래그가 계좌 절에 나온다
- placeholder 자산은 빠진다
- 카드·대출·할부가 각 부채 하위 절에 나온다
- 렌더 결과에 금액(minor) 패턴이 없다
- 두 번 다시 쓰기 시 목록 절 동등(시각 제외)
- 원장 없으면 비0 exit, 파일 미생성

### 수동 테스트 / 기획 체크

1. scratch 장부에서 통장·카드·대출 추가 → 다시 쓰기 → `status.md` 확인
2. `status.md`를 손으로 망가뜨린 뒤 다시 쓰기 → 원장과 일치
3. 파일 삭제 후 에이전트 안내대로 목록 명령 폴백이 가능한지 확인
4. 잔액 조회와 파일이 다른 역할인지(파일에 잔액 없음) 확인

## 마이그레이션과 호환성

없음. 새 파일은 옵트인 생성(명령 또는 구조 변경 훅).

## 위험과 롤백

| 위험 | 대응 |
|---|---|
| 손메모를 `status.md`에 넣음 | SPEC·스킬에 다시 쓰면 덮임을 명시. 메모는 dash MDX |
| 자동 다시 쓰기가 느림 | 구조 변경 경로만. txn마다 돌리지 않음 |
| 에이전트가 잔액으로 오해 | 1차에 금액 칸 금지 + 스킬에 잔액은 조회 명령 |

롤백: 명령·렌더·스킬 문구 제거. 데이터 이주 없음. 남은 `status.md`는 무시하면 됨.
