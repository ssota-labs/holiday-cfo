## 목표

[prd-dash-mdx-shell](/docs/planning/prds/prd-dash-mdx-shell)대로 사용자
`dash/`를 fumadocs 앱으로 교체한다. fumadocs 기본 테마·컴포넌트,
`@holiday-cfo/ui` 셸, `@holiday-cfo/blocks`를 대시보드와 MDX에 등록하고,
json-render/`spec.json`을 제거한다. 숫자는 API·bake. 결정은
[ADR-012](/docs/development/adr/adr-012).

## 영향 범위

| 영역 | 변경 |
|---|---|
| `packages/cli/templates/dash/` | vinext 템플릿 → fumadocs(Next) 템플릿. content MDX, `/dashboard`, API, `getMDXComponents` |
| `packages/cli/src/dash.ts` | scaffold/bake가 MDX·대시보드 소스를 덮지 않음. 설명 문구 갱신 |
| `packages/cli/src/cli.ts` | `dash init` 설명(vinext → fumadocs) |
| `packages/blocks` | json-render 의존 제거. MDX/대시보드용 컴포넌트·데이터 컨텍스트 export. 금액 prop 없음 유지 |
| `packages/ui` | dash/MDX에 노출할 셸 칸 re-export 또는 문서화(필요 시) |
| `plugins/*/skills/holiday-cfo/` | dash 안내를 fumadocs·MDX·API·`dash data`로 갱신 |
| `apps/docs` | 본 기획·스펙·ADR·기능 페이지 (이 PR). 구현 PR에서 스펙 이력·비전 문구 정합 |
| `AGENTS.md` | 레이아웃 설명(vinext/`spec.json` → fumadocs/MDX/API) |
| `pnpm-lock.yaml` | blocks에서 `@json-render/*` 제거에 따른 lockfile |

## 구현 순서

1. **템플릿 스파이크:** 최소 fumadocs 앱이 `dash init` 산출물로 뜨는지 확인.
   기본 테마 + content 한 페이지 + `/dashboard` 빈 골격.
2. **레지스트리:** `getMDXComponents`에 fumadocs 기본 + 선정 UI 칸 + 가계부
   블록 등록. 미등록 태그 실패.
3. **데이터:** `DataProvider`/동등 컨텍스트가 API(로컬) 또는 bake JSON(배포)을
   읽게 한다. snapshot API = bake와 동형 스키마.
4. **API 라우트:** pending / categorize 유지·이전, snapshot GET 추가. CLI 셸,
   동시 open 락 주의(기존 순차 호출).
5. **대시보드 페이지:** UI 셸 + 블록으로 기본 레이아웃(건강·런웨이·잔액·분류
   대기). json-render `Renderer` 삭제.
6. **blocks 정리:** `@json-render/*` 제거, catalog를 Zod/MDX 어휘 문서로
   이전(에이전트용 AGENTS.md + 타입).
7. **`dash init` / `dash data`:** non-clobber, 버전 pin, 첫 bake.
8. **마이그레이션 안내:** 기존 `spec.json` dash → 새 템플릿. 변환 스크립트
   또는 “다시 init + 페이지 이전” 중 하나를 고른다.
9. **스킬·템플릿 AGENTS.md:** 숫자 금지, MDX 작성, API/`dash data`, 배포 사진.
10. **테스트** 후 사용자-facing 배포는 후속 릴리스 계획으로 버전 bump + `v*`
    태그 (이 구현 PR에 public 버전을 섞지 않는다).

## 검증 계획

### 자동 테스트

- CLI dash scaffold/bake: MDX non-clobber, 스냅샷 필드, 템플릿에 json-render
  없음
- blocks: 금액 prop 스키마 거부, 미등록 타입 실패, json-render import 없음
- API 라우트 단위(가능하면): CLI mock으로 snapshot/pending/categorize
- `pnpm --filter holiday-plugin test`
- `pnpm --filter @holiday-cfo/docs check:planning`

### 수동 테스트 / 기획 체크

1. scratch에서 `dash init` → fumadocs 테마, 메모 페이지 추가, `/dashboard` 숫자
2. MDX에 `<CashRunway />`·UI Card 태그 → 렌더 확인
3. 페이지에 금액 문장 → 권위 있는 잔액 UI로 안 보임
4. 장부 변경 → `dash data` / API 재조회 → 글 유지·숫자 변경
5. 로컬 분류 클릭 → CLI 경로
6. 정적/배포 빌드에서 쓰기 불가
7. 템플릿 `package.json`에 `@json-render/*` 없음

## 마이그레이션과 호환성

- 원장 DB 마이그레이션 없음
- vinext + `spec.json` 사용자는 새 템플릿으로 이전. 병행 기간이 있으면 구현
  PR에 기한을 명시
- bake `ledger.json` 스키마는 우선 유지. 깨면 릴리스 노트
- Codex Sites/vinext 전제 문구(스킬·README)는 “읽기 전용 정적 산출”로 갱신

## 위험과 롤백

| 위험 | 완화 |
|---|---|
| Next/fumadocs 무게·Node 요구 | 로컬 조종석 전제 명시. 배포는 정적 export 스파이크로 검증 |
| MDX에 숫자 리터럴 | props 차단 + 스킬 + (선택) lint |
| UI 칸을 너무 많이 열어 레이아웃 난립 | 등록 목록을 최소 집합으로 시작, AGENTS.md에 화이트리스트 |
| Sites/vinext 사용자 경로 단절 | 스펙이 배포 전제 재정의 — 스킬·문서에 대체 호스트 안내 |
| blocks에서 json-render 제거 파급 | 템플릿·스킬·테스트를 한 PR에서 맞춤 |

롤백: 템플릿·스킬을 이전 vinext 태그로 되돌리면 된다. 원장 데이터 영향 없음.
