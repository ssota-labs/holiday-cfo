## 목표

[prd-ledger-status-session-hook](/docs/planning/prds/prd-ledger-status-session-hook)대로
유저 장부 세션이 열릴 때 `status.md` 유무를 보고, 없으면
[SPEC-ledger-status](/docs/agent/ledger-status)와 동일한 다시 쓰기로 만든다.
파일이 있으면 덮지 않는다. 실패는 soft-fail.

## 영향 범위

| 영역 | 변경 |
|---|---|
| `packages/cli` | 장부 템플릿 Cursor `sessionStart`에 ensure 스크립트(또는 기존 스크립트 확장) |
| `plugins/claude-code` | `SessionStart`에 soft-fail ensure (플러그인 루트 스크립트) |
| `plugins/codex` | 동일 |
| `apps/docs` | SPEC·기능 페이지에 세션 훅 계약, 기존 장부 수동 추가 안내 |

원장 마이그레이션 없음. `holiday status` 본문 계약은 plan-011 그대로 재사용.

## 구현 순서

1. SPEC에 이미 적힌 세션 훅 계약을 구현 체크리스트로 옮긴다
2. 공유 셸 스크립트: `.holiday/` 없으면 no-op, 모노레포면 no-op, `status.md` 있으면
   no-op, 없으면 `npx @holiday-cfo/cli@latest status` (또는 로컬 bin) soft-fail
3. Claude Code·Codex `hooks/hooks.json` SessionStart에 연결
4. Cursor 템플릿 `.cursor/hooks.json`에 항목 추가(문서 스킬 훅과 병행). init은
   **없는 파일만** 쓰므로 기존 장부 `hooks.json`은 덮지 않음 — 문서에 수동 병합
   안내
5. (호스트가 허용하면) 짧은 컨텍스트 한 줄: 생성함 / 이미 있음 / 건너뜀. 없으면
   부작용만으로도 수락
6. 스크립트·훅 경로 테스트(없으면 생성, 있으면 유지, 가드) + `check:skill` 영향 없음
   확인
7. 기능·에이전트 문서 링크 정리. 사용자-facing이면 릴리스 태그

## 검증 계획

### 자동 테스트

- 장부 scratch에 `status.md` 없음 → ensure 스크립트 → 파일 생성, 필수 절 존재
- 파일 있음 → 스크립트 후 바이트(또는 목록 절) 불변
- `.holiday/` 없음 · 모노레포 마커 → 파일 미생성, exit 0
- CLI 실패 시뮬레이션 → 스크립트 exit 0 (soft-fail)

### 수동 테스트 / 기획 체크

1. 새 `holiday init` 장부에서 Cursor/해당 호스트 세션 시작 → `status.md` 생김
2. 파일을 둔 채 세션 재시작 → 내용 유지
3. 훅 trust 거절·CLI 없는 환경 → 대화 계속, AGENTS의 수동 `holiday status` 경로 가능
4. 기존 장부(옛 `hooks.json`만 있음) → 문서의 수동 추가 안내로 동일 스크립트 연결

## 마이그레이션과 호환성

없음. 새 템플릿·플러그인 훅만. 기존 프로젝트 훅/AGENTS는 비파괴 — 사람이 추가.

## 위험과 롤백

| 위험 | 대응 |
|---|---|
| 매 세션 다시 쓰기로 손메모 소실 | **없을 때만** 생성. 있으면 no-op |
| `npx` 지연·네트워크 | soft-fail, 타임아웃 상한(문서 스킬 훅과 同계열 ≤120s) |
| Cursor만 프로젝트 로컬 훅 | 플러그인 호스트는 플러그인 갱신으로 전파. Cursor 기존 장부는 수동 |
| 훅이 잔액으로 오해 | 생성 본문은 기존 status 계약(금액 없음). 컨텍스트에도 잔액 언급 금지 |

롤백: 훅 항목·스크립트 제거. 남은 `status.md`는 무시 가능.
