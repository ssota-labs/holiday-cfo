## 목표

[prd-holiday-cfo-presence](/docs/planning/prds/prd-holiday-cfo-presence)대로
장부 대화 에이전트가 “너는 누구야?”에 **holiday 개인 CFO**로 먼저 답하게 한다.
자비스·프라이데이 분위기는 유지하되 호스트 모델 제품명으로 자기소개를 시작하지
않도록 지침을 고친다.

## 영향 범위

| 영역 | 변경 |
|---|---|
| `packages/cli/templates/ledger/AGENTS.md` | 말투 절 앞에 정체성·자기소개 예시·금지 문구 |
| `packages/cli/templates/ledger/CLAUDE.md` | `@AGENTS.md` 전에 한두 줄로 holiday 역할 명시 |
| `plugins/claude-code/skills/holiday-cfo/SKILL.md` | How you speak에 동일 정체성 계약 (기존 장부 경로) |
| `plugins/codex/skills/holiday-cfo/SKILL.md` | Claude와 같은 핵심 문장 (호스트별 서문만 유지) |
| `apps/docs` | 본 기획·스펙. 구현 PR에서 스킬 스펙 이력 한 줄 |

저장소 루트 `AGENTS.md` / `CLAUDE.md`는 **코딩 에이전트**용이므로 이번 범위에서는
장부 비서 자기소개를 넣지 않는다. 사용자 문구 용어집과의 충돌만 없는지 확인한다.

## 구현 순서

1. 스펙에 적힌 권장 예시·금지 목록을 장부 `AGENTS.md` 말투 절에 넣는다
   (이름 holiday, 첫 문장 모델명 금지, 자비스·프라이데이는 분위기만).
2. `CLAUDE.md`에 “이 폴더에서는 holiday 개인 CFO로 말한다. 상세는 AGENTS.md”를
   짧게 쓴다.
3. 양쪽 `SKILL.md` How you speak에 같은 규칙·예시를 넣는다 — 기존 장부는 여기가
   실제로 전달되는 경로다.
4. `apps/docs` 스킬 스펙 이력에 정체성 계약 한 줄을 추가한다.
5. 사용자-facing이므로 후속 릴리스에서 버전 bump와 `v*` 태그
   (`plan-release-tagged-deploys`). 이 구현 PR에 public 버전 숫자를 넣지 않아도
   되면 릴리스 PR로 분리한다.

## 검증 계획

### 자동 테스트

- `pnpm --filter @holiday-cfo/docs check:planning`
- `pnpm --filter holiday-plugin test` (스킬↔CLI 정합 — 명령 추가 없으면 회귀만)
- 선택: 세 표면에 “저는 holiday” / 모델명 선두 금지 표식이 있는지 문자열 검사
  (넣기로 하면 `plugins/claude-code/scripts`에 최소 검사)

### 수동 테스트 / 기획 체크

1. scratch에서 `holiday init` 후 `AGENTS.md` / `CLAUDE.md`에 정체성 절·예시가 보인다
2. 스킬만 갱신된 기존 장부 가정으로 “너는 누구야?” → 첫 문장이 holiday
3. “어떤 모델이야?” → 모델명을 말해도 역할은 holiday로 남는다
4. 평소 기록·잔액 답변이 매 턴 자기소개로 길어지지 않는다
5. 연극기·이모지·영화 대사가 지침 예시에 없다

## 마이그레이션과 호환성

- 원장 DB·마이그레이션 없음
- 기존 장부 `AGENTS.md`는 덮지 않음 — 스킬이 정체성 규칙의 우선 전달 경로
- 사용자가 장부 `AGENTS.md`를 직접 고친 경우는 그대로 둔다

## 위험과 롤백

| 위험 | 대응 |
|---|---|
| 호스트 시스템 프롬프트가 모델 소개를 강제 | 스킬·AGENTS에 “첫 문장은 holiday”를 못 박고 예시로 덮는다 |
| 캐릭터 연기가 과해짐 | “연극기 없음”, ✓·⚠만, 영화 대사 금지 |
| 쉬운 한국어 PR과 범위 충돌 | 이 계획은 정체성만. 도구 숨김·평이화는 별도 |

롤백: 템플릿·스킬 절을 되돌리면 된다. 데이터 영향 없음.
