## 목표

[prd-document-skills-companion](/docs/planning/prds/prd-document-skills-companion)대로
엑셀·PDF·워드·슬라이드 문서 스킬을 장부 플러그인 **companion**으로 붙인다.
본문 복사는 하지 않는다. **기본 설치는 전 호스트 skills.sh**(`npx skills add … -g -y`)이고,
세션 시작은 `npx skills update -y`(전역)다. Claude Code 마켓 플러그인은 기본 경로가 아니다.

## 영향 범위

| 영역 | 변경 |
|---|---|
| `plugins/claude-code/skills/holiday-cfo/` | setup(및 필요 시 SKILL.md 한 줄)에 companion 설치 안내 |
| `plugins/codex/skills/holiday-cfo/` | references 심링크면 Claude 쪽만; SKILL.md는 호스트별 |
| `plugins/claude-code/hooks/` | `SessionStart` → soft-fail `npx skills update -y` |
| `plugins/codex/hooks/` | 동일 (trust 전제는 스펙·README에 명시) |
| `packages/cli` init 템플릿 | Cursor `.cursor/hooks.json` `sessionStart` (없을 때만 생성) |
| `apps/docs` | 본 기획·스펙·에이전트 문서 (이 PR). 구현 PR에서 skill 스펙 이력 갱신 |
| `README.md` | 플러그인 설치 절에 companion 한 블록 (구현 PR) |

## 구현 순서

1. setup 워크플로에 skills.sh `-g -y` 네 스킬 안내만 넣는다 (Claude 포함, 마켓 기본 경로 없음)
2. 공유 갱신 스크립트(선택) — exit 0으로 soft-fail 래핑, stderr만 남김
3. Claude Code `hooks/hooks.json` SessionStart 연결
4. Codex `hooks/hooks.json` SessionStart 연결 + trust 안내
5. `holiday init`이 Cursor `sessionStart` 훅을 비파괴로 쓰게 한다
6. README·에이전트 스펙(`skill.mdx`)에 companion/훅을 짧게 반영
7. 플러그인·패키지 버전 bump + 태그 배포 (사용자-facing)

## 검증 계획

### 자동 테스트

- `pnpm --filter holiday-plugin test` (스킬↔CLI; 훅 파일이 있어도 깨지지 않을 것)
- `pnpm check:skills` — 모노레포 프로젝트 pin 불변
- `pnpm --filter @holiday-cfo/docs check:planning`
- init 스모크: 빈 디렉터리에서 hooks 파일이 생기고, 재실행 시 overwrite 없음

### 수동 테스트 / 기획 체크

1. Claude Code: 셋업의 `npx skills add … -g -y`만으로 xlsx/pdf가 잡히는지 (마켓 안내 없음)
2. 전역 설치 후 세션 재시작 → `skills update`가 no-op 또는 갱신
3. 네트워크 차단/실패 시에도 장부 명령 대화가 이어지는지
4. 모노레포에서 세션 훅을 돌려도 루트 `skills-lock.json` 해시가 변하지 않는지
5. Codex: 훅 trust 전/후 동작 차이 확인

## 마이그레이션과 호환성

- 원장 DB·마이그레이션 없음
- 기존 프로젝트: init 재실행 시에만 Cursor 훅 추가(기존 파일 유지)
- Anthropic 라이선스 본문을 MIT 트리에 넣지 않음

## 위험과 롤백

| 위험 | 완화 |
|---|---|
| 매 세션 네트워크 지연 | soft + soft-fail; 이미 최이면 다운로드 없음 |
| Codex/Cursor 훅 미지원·untrusted | 셋업 수동 안내가 1차 경로로 남음 |
| 실수로 프로젝트 lock rewrite | 스펙상 `-g`/전역만; 구현에서 cwd 프로젝트 lock 경로 미사용 |
| “latest” 비결정성 | companion는 upstream latest를 의도. 장부 계약은 holiday 버전이 고정 |

롤백: 훅 JSON·setup 문구·init 템플릿만 되돌리면 된다. 원장 데이터 영향 없음.
