>
Postgres 어댑터를 넣기 직전, SQLite 스토어를 통째로 복사·변환하려다 멈췄다.
가정을 믿지 않고 **줄을 셌다.** 약 1,486줄 중 SQLite에만 묶인 것은 여덟 줄이었다.

| 구문 | 횟수 |
|---|---|
| `GLOB` | 3 |
| `PRAGMA` | 3 |
| `INSERT OR IGNORE` | 1 |
| `BEGIN IMMEDIATE` | 1 |
| **합계** | **8** |

구조적 차이는 `node:sqlite`가 동기라는 점뿐이고, 그건 **드라이버**를 나누라는
뜻이지 스토어를 하나 더 만들라는 뜻은 아니다.

그래서 `@holiday-cfo/store-sql`이 `LedgerStore`를 **하나만** 구현한다. 엔진
패키지(`store-sqlite` / `store-postgres`)는 드라이버·스키마·마이그레이션만 갖는다.
여덟 곳은 `Dialect`로 접는다.

`GLOB`이 공유 SQL에 남은 이유는 SQLite-ism 유출이 아니다. 계정 접두 매칭은
대소문자를 구분해야 하는데, SQLite `LIKE`는 구분하지 않는다. Postgres `LIKE`는
구분한다. 같은 뜻, 다른 철자 — 드라이버가 번역한다.

옮긴 뒤 적합성 스위트 18개가 그대로 통과했다. 스위트가 없었으면 “아마 같을 것”이
최선이었다.
</Decision>
