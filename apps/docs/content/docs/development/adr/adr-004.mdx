  ]}
>
“지금 쓸 수 있는 돈인가?”는 계정 코드 접두어로 짐작하지 않는다.
계정에 **`account.cash` 플래그**를 켠다. 옛 `Assets:Bank*` 관습은 마이그레이션이
한 번 채워 주고, `account add`와 `cashflow`는 현금 표시가 없는 자산 계정을
경고로 알린다.

이 작업을 하다 **더 큰 버그**를 찾았다. 마이그레이션이 `init` 할 때만 돌아가고 있었다.
이미 있는 장부를 새 바이너리로 열면 없는 컬럼을 읽어 죽었다. `ledger.db`는 사용자가
커밋해 두는 파일이고 CLI는 따로 업데이트되니, 한 번만 올려도 **기존 장부가 전부
안 열렸을** 수 있다. 새 DB만으로 테스트하면 못 본다. 그래서 장부를 열 때
마이그레이션이 돌아가게 바꿨다.
</Decision>
