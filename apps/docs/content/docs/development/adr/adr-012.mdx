사용자 장부 프로젝트의 `dash/`는 **fumadocs(Next) 앱**이다.
`holiday dash init`이 그 템플릿을 복사한다. holiday 레포의 `apps/docs`와는
별 제품 표면이다.

채택:

1. **fumadocs 기본 컴포넌트·테마** — 메모·기록 MDX 페이지의 기본 경험
2. **`@holiday-cfo/ui` 셸** — 대시보드 라우트와 블록이 기존처럼 사용. MDX
   레지스트리에도 필요한 UI 칸을 등록해 메모 페이지에서 태그로 쓴다
3. **`@holiday-cfo/blocks` 가계부 블록** — 대시보드와 MDX 양쪽에 등록.
   금액·쓰기 액션 prop 없음
4. **json-render / `spec.json` 제거** — 배치는 MDX와 대시보드 페이지
5. **API** — 로컬에서 잔액·현금흐름·건강·분류 대기를 CLI 경유 API로 읽기/쓰기.
   배포 사진은 bake된 읽기 전용 데이터 + 쓰기 API 없음
6. **대시보드 별 라우트** — 숫자 조종석은 `/dashboard`(또는 동등). 메모는
   content MDX 트리

[ADR-008](/docs/development/adr/adr-008)의 로컬 조종석 / 배포 사진 경계는
그대로다. Vision의 “웹 대시보드를 제품으로” 거부와도 맞다 — 본체는 CLI,
화면은 스냅샷·기록 보조다.
</Decision>
