## 한 줄

플러그인을 켠 **유저 각자의 장부 프로젝트** 안에 엑셀·PDF·워드·슬라이드용
문서 스킬이 같이 있고, 그 프로젝트에서 세션이 열릴 때마다 최신으로 맞춘다.

## 문제

첫 세션과 이후 수집에서 CSV·엑셀 내보내기, 홈택스·신고서 PDF를 자주 본다.
지금 장부 플러그인은 “파일을 직접 읽어 옮기라”고만 안내할 뿐, 문서 형식을
다루는 전용 스킬을 기본으로 붙이지 않는다.

그 결과:

- 호스트마다 설치 경로가 달라 어떤 사람은 문서 스킬 없이 장부만 켠다
- 예전에 깐 문서 스킬이 오래되면 같은 파일을 어제와 다르게 다룬다
- 장부 플러그인 안에 문서 스킬 본문을 복사해 넣으면 라이선스·갱신·컨텍스트
  부담이 한꺼번에 커진다
- 사용자 홈 전역에만 깔면 장부 폴더와 버전이 어긋나기 쉽고 프로젝트마다
  재현하기 어렵다

문서 스킬은 Anthropic이 관리하는 별도 제품이다. 장부 플러그인에 복사해 넣지 말고,
**그 유저의 장부 프로젝트 폴더**에 skills.sh로 설치·갱신하는 편이 맞다.
Claude Code도 예외가 아니다.

이 계약이 다루는 대상은 holiday 모노레포가 아니다. 플러그인을 설치해 쓰는 **유저별
프로젝트**(보통 `.holiday/`가 있는 장부 디렉터리)다.

## 무엇을 저장·보여 주는가

장부 DB에 새 칸을 만들지 않는다. 유저 프로젝트 폴더에 남는 것은 설치·갱신 흔적이다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 문서 스킬 설치 안내 | “이 장부 폴더에 엑셀·PDF 스킬을 먼저 깔아 주세요” | 첫 세션에서 빠진 설치를 채운다 |
| 프로젝트 스킬 폴더 | 장부 프로젝트의 `.agents/skills/` | 그 프로젝트만의 문서 스킬 복사본 |
| 프로젝트 잠금 파일 | 장부 프로젝트의 `skills-lock.json` | 이 폴더에 깐 버전 지문 |
| 세션 시작 갱신 | 그 프로젝트에서 열릴 때 최신인지 확인 후 필요할 때만 받음 | 사람이 `update`를 기억하지 않아도 된다 |

잠금 파일은 알아서 바뀌지 않는다. 그 프로젝트에서 세션 시작 훅이 갱신 명령을
돌릴 때 성공한 스킬의 해시를 그 프로젝트 잠금 파일에 다시 쓴다.

## 목표

1. 첫 세션(셋업)에서 문서 스킬 설치를 빠뜨리지 않게 안내한다.
2. 기본 설치는 모든 호스트에서 **프로젝트 폴더** skills.sh 설치다
   (`npx skills add … -y`, `-g` 없음). Claude Code도 같다.
3. Cursor·Codex·Claude Code 모두, 그 장부 프로젝트에 이미 깐 스킬은 세션
   시작 때 skills.sh **프로젝트 갱신**으로 최신을 맞춘다.
4. 장부 플러그인에는 문서 스킬 본문을 넣지 않는다. 안내·훅·래퍼만 둔다.

## Non-goals

- 문서 스킬 폴더를 `holiday-cfo` 플러그인 skills/에 복사·벤더링
- 사용자 홈 전역(`-g`) 설치를 기본 경로로 두기
- Claude Code 전용으로 Anthropic 마켓 `document-skills` 설치를 기본 경로로 두기
- holiday 모노레포 개발용 잠금·스킬 트리를 이 계약으로 관리하기
- 마켓플레이스 `dependsOn` 같은 호스트 미지원 필드를 새로 발명
- OCR·세액 공식·은행 고정 파서 — 기존 “관측값만 / 에이전트가 importer” 경계를 바꾸지 않음
- pptx·docx가 장부 핵심 워크플로가 되게 만들기 (설치·갱신 대상에는 포함)

## 비기능

- 세션 시작 갱신이 실패해도 대화를 막지 않는다 (네트워크·미설치·권한).
- 갱신 범위는 **현재 연 장부 프로젝트**의 skills.sh 프로젝트 스킬이다
  (`skills update -p`). 사용자 홈 전역 잠금은 건드리지 않는다.
- Codex 플러그인 훅은 호스트가 신뢰(trust)를 요구하기도 한다. 신뢰하기 전에는
  안내 문구만으로도 설치 경로가 잡혀야 한다.
- Cursor는 holiday 마켓 플러그인과 배포 경로가 다르다. 세션 훅은 init이 쓰거나
  문서에 적는 방식으로 맞춘다.

## 성공 지표

- 셋업을 따르는 사용자는 엑셀·PDF 문서 스킬 없이 첫 import에 들어가지 않는다
- 셋업 기본 설치 명령이 프로젝트 skills.sh(`npx skills add … -y`)이다
- 유저 장부 프로젝트에서 세션을 열면, 그 프로젝트에 깐 문서 스킬이
  원격과 다를 때 받아 온다
- 갱신 실패 시에도 장부 대화는 이어진다
- 장부 플러그인 디렉터리에 Anthropic 문서 스킬 본문이 없다

## 결과 계약

문서 스킬은 companion이다. 장부 스킬이 원장 결과(수집·확정·잔액 대조)를
책임지고, companion는 파일 형식만 다룬다. 설치·갱신 계약의 기계 표현은
[SPEC-document-skills-companion](/docs/agent/document-skills-companion)에 둔다.
