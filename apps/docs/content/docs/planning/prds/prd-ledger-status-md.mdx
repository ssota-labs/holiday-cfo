## 한 줄

장부에 있는 **계좌와 부채**를 마크다운 한 장에 적어 두면, DB를 매번 열지 않아도
사람과 채팅 에이전트가 바로 훑어본다.

## 문제

“내 통장이 뭐가 있지?”, “카드·대출이 뭐가 있지?”를 알려면 지금은 장부 조회를
매번 돌려야 한다. 에이전트도 세션마다 목록 명령을 다시 친다.

틀린 방식은 아니다. 다만 이렇게 불편하다.

- 사람은 에디터에서 파일 하나만 열어 보고 싶은데, 조회 출력은 잠깐 보이다 사라진다
- 에이전트도 방향만 잡을 때 CLI를 여러 번 돌린다
- 대시보드 메모·조종석은 숫자 화면용이라, 목록만 빠르게 보기와는 결이 다르다

진실은 여전히 장부다. 마크다운은 **읽기용 요약**이고, 손으로 고쳐 진실을 바꾸는
칸이 아니다.

## 무엇을 저장·보여 주는가

장부에 새 사실 표를 만들지 않는다. 이미 있는 계정·카드·대출·할부를
**읽기 쉬운 목록**으로만 다시 쓴다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 현황 파일 | 장부 폴더의 `status.md` | 에디터·에이전트가 같은 파일을 연다 |
| 갱신 시각 | 2026-07-22 17:00 | 얼마나 지난 목록인지 바로 본다 |
| 계좌 목록 | KB 입출금(쓸 수 있는 돈), 카카오 통장 | “돈이 어디에 있나”를 한눈에 |
| 부채 목록 | 신한 카드, 주담대, 냉장고 할부 | “어떤 빚이 있나”를 한눈에 |
| 짧은 표지 | 현금 표시 / 카드·대출·할부 구분 | 이름만으로 헷갈릴 때 종류를 짚는다 |

1차 범위는 **나열**이다. 잔액·현금흐름·마감 숫자는 이 파일에 넣지 않는다.
그 숫자는 지금처럼 잔액 조회·현금흐름·대시보드에서 본다.

## 목표

1. 장부 폴더에 현황 마크다운을 두고, 계좌·부채 이름을 파일만으로 읽는다
2. 채팅 에이전트는 방향을 잡을 때 그 파일을 먼저 읽고, 없을 때만 목록 조회로
   넘어간다
3. 계정·카드·대출·할부를 바꾼 뒤 같은 내용으로 다시 써 목록을 맞춘다
4. 파일을 손으로 고쳐도 진실은 바뀌지 않는다. 다시 쓰면 장부 내용으로 덮인다

## Non-goals

- 현황 파일을 장부 진실(SoR)로 두기
- 잔액·순자산·현금흐름·유동·비유동 합계를 이 파일에 싣기 (1차)
- 수익·비용 계정 차트 전체를 나열하기 (1차 — 계좌·부채 중심)
- 대시보드 MDX·숫자 스냅샷을 이 파일로 대체하기
- 사람이 메모한 자유 문장을 이 파일에 섞어 보존하기 (다시 쓰면 사라짐)

## 비기능

- 파일은 장부 프로젝트 안에 둔다. 원장 DB와 같이 private 저장소 전제다
- 금액 칸을 1차에 넣지 않으므로, 오래된 목록이어도 틀린 잔액으로 오해할 위험은
  없다. 다만 **빠진·옛 이름**은 남을 수 있어 갱신 시각을 꼭 적는다
- 다시 쓰기는 멱등이다. 같은 장부면 같은 구조의 마크다운이 나온다

## 성공 지표

1. 현황 파일을 열면 계좌 이름과 부채(카드·대출·할부) 이름이 보인다
2. 갱신 시각이 파일 안에 있다
3. 계정·카드·대출·할부를 추가한 뒤 다시 쓰면 새 이름이 목록에 나타난다
4. 파일만 고쳐서는 장부 계정·스케줄이 바뀌지 않는다
5. 에이전트 안내에 방향 잡을 때 현황 파일을 먼저 읽는다는 내용이 들어 있다
