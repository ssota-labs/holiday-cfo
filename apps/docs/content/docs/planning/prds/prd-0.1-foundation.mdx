## 한 줄로

1인이 **통장·카드·할부·정기·대출**을 로컬 장부(SQLite)에 복식으로 남긴다.
**현금이 부족한 날(SHORT)**을 CLI로 미리 본다. 에이전트는 같은 CLI를
기계가 읽는 계약(`--json`, exit code)으로 호출한다.

## 문제

카드 청구서, 할부표, 구독·급여, 대출 상환이 **명세서와 머릿속에만** 있으면
“다음 달에 현금이 모자라는가?”를 믿기 어렵다. 스프레드시트는 복식 균형을
지키지 못한다. 은행·카드 API나 OCR에 기대면 시작부터 무겁다.

대략 이런 칸이 비어 있었다.

- 내 컴퓨터에만 두는 **한 장부**
- 차변이 맞아야만 들어가는 **전표**
- 카드·할부·정기·대출처럼 **아직 확정되지 않은 예정**을 장부와 분리해 두는 방식
- 캡처에서 읽은 건을 **분류 대기 → 승인/반려**로 넣는 흐름
- “잔액이 맞는지 / 감사가 깨지지 않았는지”를 **한 번에 확인**하는 명령

foundation(`0.1`)이 이 기반을 깐다. 이후 범위는 이 PRD에 넣지 않는다.

## 무엇을 저장·보여 주는가

### 1) 장부 뼈대

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 기능통화 | KRW | 잔액·현금흐름의 기준 통화 |
| 타임존·마감 단위 | Asia/Seoul, 일 | 날짜·마감 경계를 맞추기 위해 |
| 계정 | `Assets:Cash:Bank`, `--cash` | 어디에 돈이 있는지, 현금흐름 시작 잔액에 쓸지 |
| 로컬 DB 경로 | `.holiday/ledger.db` | 같은 폴더에서 CLI로 바로 기록 |

`init`으로 장부를 만든다. `account add` / `list`로 차트를 둔다.

### 2) 전표(확정·대기)

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 복식 다리 | 현금 −65,000 / 식비 +65,000 | 합이 0이 아니면 거절 (불균형 금지) |
| 확정 / 대기 | posted / draft | 대기 건은 잔액·현금흐름에 넣지 않음 |
| 잔액 조회 | `balance` | 계정별 무게(weight) 잔액 |

### 3) 예정 스케줄 (장부에 미리 전기하지 않음)

청구·할부·정기·대출은 **예측**이다. 금리·일정이 바뀌어도 과거 확정 전표를
건드리지 않으려고, 예정은 확정 전표와 따로 둔다.

| 사람이 읽는 이름 | 무엇을 남기는가 | 왜 필요한가 |
|---|---|---|
| 카드 | 결제일·청구 주기 | 다음 출금일을 현금흐름에 반영 |
| 할부 | 회차·원금·관측 수수료 | 남은 회차 출금을 투영 (`--fees`는 관측값만) |
| 정기지출 | 구독·고정비 주기 | 매월(또는 주기) 나가는 돈 |
| 정기수입 | 급여·이체 주기 | 들어오는 돈·정산 |
| 대출 | 상환 일정·대사 | 원리금 예정과 실제를 맞춤 |

### 4) 수집·검토

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 수집(대기) | 캡처에서 읽은 건 → draft | OCR/은행 API 없이 에이전트가 초안만 넣음 |
| 중복 차단 | 같은 이미지·외부 키 | 두 번 넣지 않기 |
| 승인 / 반려 | accept / reject | 사람 확인 후에만 확정; 반려도 중복 기억에 남김 |

### 5) 환율·마감·현금흐름·감사

| 사람이 읽는 이름 | 무엇을 보여 주는가 | 왜 필요한가 |
|---|---|---|
| 수동 환율 | 날짜별 관측 환율 | 자동 수집 없이 외화 다리를 맞출 때 |
| 잔액 대조·마감 | assert / close | 통장·카드 잔액과 장부를 맞추고 기간을 닫음 |
| 현금흐름 | 일자별 잔액, **SHORT** | 스케줄 반영 후 잔액이 음수가 되는 날 |
| 감사 체인 | `verify` OK | 장부가 깨지지 않았는지 확인 |

## 목표

- 로컬 SQLite 장부 + CLI (`@holiday-cfo/cli`)로 위 범위를 모두 다룬다
- 계정·전표·카드·할부·정기·대출·환율·수집/검토·대조/마감·현금흐름·감사를 제공한다
- 정책(POLICY-001…016)을 테스트에 고정한다
- 에이전트 스킬이 CLI를 `--json`으로 shell-out 한다

## Non-goals

- OCR / 은행·카드 API 연동
- 환율 자동 수집
- Postgres를 CLI의 기준 저장소로 쓰기
- 웹 UI·여러 사람이 쓰는 클라우드 장부
- 할부수수료·이자 공식을 추정하기 (관측값만)

## 비기능

- Node ≥ 24, `node:sqlite` (네이티브 애드온 없음)
- 금액 JSON은 소수 문자열(정밀도 유지). 내부는 정수 minor unit
- 이미 적용된 마이그레이션은 고치지 않는다 (해시 불변)

## 성공 지표

1. `verify`가 OK를 낸다
2. `cashflow`가 잔액 부족 날짜에 **SHORT**를 표시한다
3. 정책↔테스트 링크가 `pnpm --filter @holiday-cfo/docs check:rules`를 통과한다
4. CLI 스펙 스냅샷 `0.1.0`이 문서에 존재한다
