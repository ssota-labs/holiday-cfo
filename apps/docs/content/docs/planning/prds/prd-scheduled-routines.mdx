## 한 줄

장부가 생긴 뒤부터는 **매일·매주·매월** 정해진 시각에 비서가 먼저 말을 건다.
오늘 쓴 것을 받거나, 어제·오늘·다음 주·지난달 상황을 짧게 알려 준다.

## 문제

스킬에 장부 리듬(저녁 수집, 아침 리포트, 주간 점검, 월 마감 준비)이 적혀 있어도
사람이 기억해 두고 먼저 물어야 한다. 셋업이 끝나면 “자동화 걸어둘까요?”라고만 묻고
등록은 사용자에게 넘긴다. 스케줄은 비어 있고, 매일·매주·매월 안내도 오지 않는다.

원하는 리듬은 다음과 같다.

- **매일 저녁** — 오늘 쓴 것을 올려 달라고 묻는다
- **매일 아침** — 어제 요약과 오늘 현금 이벤트를 알려 준다
- **일요일 밤·월요일 아침** — 이번 주 지출과 다음(또는 이번) 주 이벤트를 나눈다
- **말일 전·매월 1일** — 이번 달 현황과 지난달 마감 제안을 나눈다

말투는 개인 비서 톤을 쓴다. 보고를 먼저 하고, 다음 행동은 한 줄로 붙인다. 과장과 이모지는 쓰지 않는다.

## 무엇을 저장·보여 주는가

장부 DB에는 새 칸을 두지 않는다. 스케줄 항목은 호스트(에이전트 앱) 쪽에 생긴다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 매일 저녁 수집 | 매일 21:00 “오늘 쓴 거 있으면 올려주세요” | 하루 지출을 그날 밤에 받는다 |
| 매일 아침 리포트 | 매일 08:00 어제 요약 + 오늘 이벤트 | 새 거래를 묻지 않고 보고만 한다 |
| 일요일 밤 주간 | 매주 일 21:30 이번 주 지출 + 다음 주 이벤트 | 주를 닫아 보고한다 |
| 월요일 아침 주간 | 매주 월 08:15 이번 주 현금흐름 | 주 초 브리핑만 짧게 |
| 말일 전 월간 | 매월 27일 21:00 이번 달 현황 + 마감 준비 | 마감 전에 할 일을 알린다 |
| 1일 월간 | 매월 1일 09:00 지난달 요약 + 마감 제안 | 확인 없이 달을 닫지 않는다 |

시간대 기본값은 Asia/Seoul이다. 사용자가 고른 주기만 등록한다.

## 목표

1. 셋업이 끝난 뒤, 동의한 주기를 **에이전트가 호스트 도구로 직접** 등록한다.
   사용자에게 수동 복붙만 시키지 않는다.
2. Claude Code와 Codex는 각각 그 호스트의 스케줄 도구·대화를 쓴다.
3. 여섯 주기마다 역할·말투·하지 말 것을 스킬과 문서 한곳에 모아 둔다.
4. 마감은 제안만 한다. 사람이 잔액·승인을 주기 전에는 달을 닫지 않는다.

## Non-goals

- CLI 안에 자체 cron·알림 서버를 만들지 않는다
- Cursor 전용 자동화를 이번 범위의 필수 경로로 두지 않는다 (안내만 가능)
- 팩스 수집용 자동화와 섞지 않는다
- 투자·절세 조언, 잔액·이자 추측
- 사용자가 거절한 주기까지 기본으로 전부 등록

## 비기능

- 스케줄러는 호스트 소유다. 장부 CLI는 스케줄을 모른다.
- 장부가 로컬 파일인 호스트라면 **로컬 프로젝트에서** 돌릴 수 있어야 한다.
- 등록 도구가 없거나 로그인이 안 되면 막지 말고, 수동 등록 문구를 건넨다.

## 성공 지표

- 셋업이 끝나면 동의한 주기가 호스트 스케줄 목록에 보인다
- 저녁 수집과 아침 리포트가 서로의 역할을 섞지 않는다
- 월 1일 실행이 사람 확인 없이 달을 닫지 않는다
- 말투가 비서 보고체(존댓말, 다음 행동 한 줄)를 지킨다

## 결과 계약

스킬·에이전트 문서에 여섯 주기 프롬프트와 호스트별 등록 절차를 둔다.
셋업 마지막 단계에서 동의한 주기를 도구로 등록한다.
