## 한 줄로

홈택스·신고서 PDF에 찍힌 **종합소득세 세액계산 숫자**를 통장 전표와 따로
장부(로컬 DB)에 **사실로 남겨**, 연도별로 다시 볼 수 있게 한다.

## 문제

`v0.2.4`까지 세금은 **통장에서 나간 돈 / 비용으로 잡은 돈**으로만 다룬다.
세금 비용 전표가 그 예다. “올해 세금으로 얼마를 냈나?”는 답할 수
있지만, 국세청에 제출한 신고서의 다음 숫자는 **장부에 없다**.

- 소득금액은 얼마였나
- 과세표준은 얼마였나
- 산출세액·세액공제·기납부세액·납부할세액은 얼마였나

지금은 연도마다 PDF나 홈택스 화면을 다시 열어 보거나, 스프레드시트에 손으로 옮겨
적어야 한다. 신고서 숫자는 예측이 아니라 **이미 확정된 사실**이다. 현금·부채
기록에서 장부를 기준 원천(SoR)으로 삼듯 신고서 숫자도 로컬 DB에 남겨야 한다.
`v0.3.0`에서 이 공백을 메운다.

## 무엇을 저장하는가

신고 한 건은 **표지 정보**와 **세액표 숫자**, 두 부분으로 이루어진다.

### 1) 신고 표지

“이번 신고가 무엇인지”를 적은 표지다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 신고서 종류 | 종합소득세 | 나중에 다른 세목을 열거나 구분할 때 |
| 귀속연도 | 2025 | “몇 년도 소득에 대한 신고인가” |
| 신고일 | 2026-06-22 | 언제 제출했는지 |
| 신고 차수 | 1차(최초), 2차(수정)… | 수정신고를 덮어쓰지 않고 이력으로 남기기 위해 |
| 메모·원본 파일 | 선택 | 사람이 알아볼 메모, PDF 경로 등 |

### 2) 세액표 숫자

종합소득세 **세액계산서의 각 칸에 적힌 값**을 그대로 옮긴다.
세율을 적용하거나 세액을 다시 계산하지 않는다.
화면을 보고 옮겨 적은 **관측값**만 받는다.

1차 범위에서 다루는 대표 칸(종합소득 열 기준):

| 사람이 읽는 이름 | 신고서에서 보는 것 | 예시(원) |
|---|---|---|
| 소득금액 | 종합소득금액 | 22,144,107 |
| 소득공제 | 소득공제 합계 | 2,350,580 |
| 과세표준 | 과세표준 | 19,793,527 |
| 세율 | 적용 세율(%) | 15.0 |
| 산출세액 | 산출세액 | 1,709,029 |
| 세액감면·세액공제 | 감면·공제액 | 70,000 |
| 결정세액 | 결정세액(합계) | 1,639,029 |
| 가산세 등 | 가산세·추가납부 | 107,570 |
| 기납부세액 | 이미 낸 세금 | 563,323 |
| 납부할세액 | 기한 내 납부할 세액 | 1,183,276 |

농어촌특별세처럼 **별도 열**이 있으면 같은 방식으로 칸별 숫자를 함께 남긴다.
칸이 비어 있으면 그 칸은 저장하지 않는다. 다만 종합소득 열에서는
**소득금액·과세표준·납부할세액**은 꼭 있어야 한다 — 없으면 “신고서를 남긴다”고
보기 어렵다.

### 3) 수정신고는 덮어쓰지 않는다

최초 신고를 지우고 고치는 것이 아니라, **새 차수 신고를 하나 더 추가**한다.
이전 차수는 “대체됨”으로 표시한다. 연도별로 “지금 유효한 신고”와 “예전 신고
이력”을 모두 볼 수 있어야 한다.

### 4) 통장 납부와는 별개

신고서를 장부에 넣는 동작은 **전표를 만들지 않는다**.
실제로 세금을 낼 때는 지금처럼 **세금 비용·납부 전표**로 따로 기록한다.

| 질문 | 어디에 답이 있나 |
|---|---|
| 통장에서 세금으로 얼마가 나갔나 / 비용으로 얼마를 잡았나 | 전표·세금 계정 |
| 신고서에 소득금액·과세표준·납부할세액이 얼마였나 | 신고서 기록 (이 PRD) |

## 목표

- 위 표지 + 세액표 숫자를 로컬 장부에 저장한다
- 1차 범위는 대한민국 **종합소득세** 세액계산 구간
- 등록·목록·조회로 연도별 신고를 다시 볼 수 있다
- 수정신고는 이력을 남긴다 (덮어쓰기 금지)
- **세액·공제 공식을 추정하지 않는다** — 신고서에 찍힌 숫자만

## Non-goals

- 홈택스 로그인·API·PDF OCR로 자동 긁어오기
- 장부에서 세액을 다시 계산하거나 세율표를 유지하기
- 지방소득세·부가가치세·원천세 전용 화면·양식 (종류만 열어 두고 1차는 종합소득세)
- 신고서 숫자와 세금 전표를 자동으로 맞춰 주거나 전표를 대신 만들어 주기
- 웹 UI·여러 사람이 쓰는 클라우드 장부

## 비기능

- 금액은 원 단위 정수(소수 오차 없이)
- 이미 적용된 DB 변경은 고치지 않고 새 변경만 추가한다
- 기존 장부 아키텍처의 의존 방향을 지킨다

## 성공 지표

1. 2025년 종합소득세 샘플 세액표를 넣으면, 조회 결과에 **소득금액·과세표준·납부할세액**이
   신고서와 같은 숫자로 보인다
2. 같은 종류·같은 연도·같은 차수를 다시 넣으면 **거절**된다 (중복 방지)
3. 수정신고를 넣으면 **차수가 올라간 새 기록**이 생기고, 이전 기록은 “대체됨”으로 남는다
4. 신고서를 넣어도 **세금 전표는 생기지 않는다**
