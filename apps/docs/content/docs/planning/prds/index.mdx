새 개발 주기마다 PRD를 **추가**한다. 끝나면 `status: done`으로 남긴다.
개별 PRD는 사이드바에 펼치지 않는다 — 아래에서 고른다.

| | ID | Period | Status | 요약 |
|---|---|---|---|---|
| <DocKind kind="PRD" /> | [prd-001](/docs/planning/prds/prd-001) | 0.1 foundation | done | 로컬 CLI 원장·스케줄·수집·마감·현금흐름 기반 |
| <DocKind kind="PRD" /> | [prd-002](/docs/planning/prds/prd-002) | continuous | active | 기획 문서가 코드보다 먼저 존재하는 개발 절차 |
| <DocKind kind="PRD" /> | [prd-003](/docs/planning/prds/prd-003) | v0.3.0 (기준 v0.2.4) | active | 세금 신고서(소득금액·과세표준·세액)를 원장 SoR로 보관 |

활성 PRD가 없으면 Vision과 Features가 현재 제품의 기준이다. ID는 소문자 짧은 키(`prd-001`)다.
다음 주기에 `prd-NNN`을 추가하고 `stories[]`로 `us-NNN`을 연결한다.
