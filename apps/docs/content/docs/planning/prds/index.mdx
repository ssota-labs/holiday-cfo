새 개발 주기마다 PRD를 **추가**한다. 끝나면 `status: done`으로 남긴다.

| ID | Period | Status | 요약 |
|---|---|---|---|
| [prd-0.1-foundation](/docs/planning/prds/prd-0.1-foundation) | 0.1 foundation | done | 로컬 CLI 원장·스케줄·수집·마감·현금흐름 기반 |
| [PRD-docs-first-development](/docs/planning/prds/prd-docs-first-development) | continuous | active | 기획 문서가 코드보다 먼저 존재하는 개발 절차 |
| [PRD-tax-return-sor](/docs/planning/prds/prd-tax-return-sor) | 0.3 tax | active | 세금 신고서(소득금액·과세표준·세액)를 원장 SoR로 보관 |

활성 PRD가 없으면 Vision과 Features가 현재 제품의 기준이다. 다음 주기에 `planning/prds/prd-…`를 추가하고 `stories[]`로 US를 연결한다.
