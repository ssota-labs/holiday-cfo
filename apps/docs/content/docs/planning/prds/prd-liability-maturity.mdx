| 기준 (현재 배포) | 부채는 카드·할부·대출·미지급 등 상품별 계정만 있음 |
| 목표 | 마감·질의 때 유동·비유동 합계를 함께 본다 |

## 한 줄

기준일을 잡으면 지금 남은 빚을 **앞으로 12개월 안에 갚을 분(유동)** 과
**그 뒤(비유동)** 로 나눠서 보여 준다. 계정 이름은 다시 짜지 않는다.

## 문제

장부에는 부채 잔액이 있다. 카드·할부·대출·미지급처럼 **어떤 빚인지**도 보인다.
“그중 1년 안에 나갈 돈은 얼마인가?”에는 아직 답이 없다.

흔히 이렇게 묻는다.

- 지금 유동부채가 얼마야?
- 이번 달 마감 기준으로 단기 빚과 장기 빚이 어떻게 나뉘어?

지금은 대출·할부 스케줄을 손으로 더하거나 통장·앱을 다시 연다.
잔액 조회·마감 보고 어디에도 그 칸이 없다.

## 무엇을 보여 주는가

새 “사실 표”를 만들어 두지 않는다. **기준일 잔액 + 이미 있는 상환·할부 예측**을
써서 그 순간 구분을 낸다. 전표를 나누거나 계정을 옮기지 않는다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 기준일 | 2026-07-31 | 같은 빚도 날짜가 바뀌면 유동·비유동 비중이 달라진다 |
| 유동부채 합계 | 4,200,000원 | 앞으로 12개월 안에 갚을 것으로 보는 원금 |
| 비유동부채 합계 | 86,000,000원 | 그 뒤에 남는 원금 |
| 부채 총액 | 90,200,000원 | 위 둘의 합. 장부 부채 잔액과 맞는지 검산 |
| 계정별 구분 | 주담대 유동 6,000,000 / 비유동 84,000,000 | “어느 빚이 단기인가”를 짚을 때 |

상품마다 기대하는 결과:

| 부채 종류 | 나누는 방식 |
|---|---|
| 카드 이용대금·미지급 | 전액 유동 |
| 할부 | 남은 회차 중 12개월 안 원금 = 유동, 나머지 = 비유동 |
| 대출 | 상환 예측의 원금 줄 중 12개월 안 = 유동, 나머지 = 비유동 |
| 예측이 없는 부채 잔액 | 전액 유동으로 본다 (단기 압박을 과소평가하지 않으려고) |

## 목표

- 잔액 조회와 월 마감 결과에서 유동·비유동 합계를 바로 본다
- 사용자가 “유동부채 얼마야?”라고 물으면 같은 숫자로 답한다
- 계정 차트에 유동/비유동 폴더를 만들지 않는다
- 구분 때문에 정정·재분류 전표를 만들지 않는다

## Non-goals

- 자산의 유동·비유동 구분 (1차 범위 밖)
- 계정 코드를 `유동부채` / `비유동부채`로 재편
- 구분 결과를 원장 전표나 별도 마스터로 영구 저장 (1차에서는 계산·표시만)
- 이자·수수료를 “유동부채”에 따로 쌓기 — **원금**만 나눈다
- 카드사·대출사 앱과 칸 단위 대사 (잔액 대조는 기존 흐름 그대로)

## 비기능

- 금액은 기존과 같이 최소 화폐 단위 정수로 다룬다. 반올림·허용오차로 맞추지 않는다
- 상환·할부 예측은 지금처럼 원장 밖에 둔다. 구분은 그 예측을 **읽어** 만든다
- 장부 부채 잔액이 진실이다. 예측 합이 잔액과 어긋나면 잔액을 기준으로 삼는다.
  예측으로 나눈 뒤 남는 차이는 유동에 붙인다

## 성공 지표

1. 기준일을 주면 유동·비유동·총액이 나오고, 유동+비유동 = 부채 총액이다
2. 카드·미지급만 있는 장부에서는 전액이 유동으로 나온다
3. 남은 기간이 12개월을 넘는 대출·할부에서는 비유동이 0이 아니다
4. 월 마감(미리보기 포함) 결과에 같은 합계가 붙는다
5. 계정 목록·코드에 유동/비유동 루트가 생기지 않는다
6. 구분 조회만으로 새 전표가 생기지 않는다
