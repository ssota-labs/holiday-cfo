## 한 줄

릴리스마다 **장부에 복사해 둔 파일**이 어떻게 바뀌었는지 플러그인 안에 정리해 두고
에이전트가 그 목록을 읽어 **필수·선택**을 나눠 맞춘다. 손댄 문장은 확인 없이 지우지 않는다.

## 문제

장부를 돌리는 쪽에는 갱신 경로가 여러 겹이다.

| 무엇이 | 어떻게 새 버전이 오나 | 지금 |
|---|---|---|
| 문서 스킬(엑셀·PDF 등) | `skills.sh` / 세션 시작 `skills update -p` | 갱신됨 |
| holiday-cfo 플러그인 스킬 | 호스트 플러그인 업데이트 | 갱신됨 |
| 장부 `AGENTS.md` · `CLAUDE.md` | `holiday init`이 **없을 때만** 복사 | 한 번 생기면 그대로 |
| `dash/` 앱·핀 버전 | `holiday dash init`이 **없을 때만** 복사 | 한 번 생기면 그대로 |

스킬만 올려도 옆 지침·대시보드는 예전 문장으로 남을 수 있다. 사용자가 고쳐 둔 문장을
아무 말 없이 덮으면 그 수정도 사라진다.

장부는 원래 **에이전트가 운전**한다. “CLI가 파일을 통째로 갈아끼운다”보다
**플러그인에 버전별 변경 쪽지를 두고 에이전트가 읽으며 맞추는 길**이 맞다.
다만 변경마다 무게가 같지 않다. 반드시 반영할 것과 권장만 할 것이 갈린다.

## 무엇을 저장·보여 주는가

원장 전표 칸은 늘리지 않는다. 정보는 두 곳에 남는다.

**1) 플러그인 — 버전별 변경 목록 (사람이 읽는 쪽지)**

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 버전 | `0.4.2` | 어느 릴리스의 변경인지 |
| 항목 이름 | “정체성 자기소개 절” | 한 줄로 무엇인지 |
| 무게 | 필수 / 선택 | 반드시인지 권장인지 |
| 대상 파일 | 장부 `AGENTS.md` | 어디를 손대는지 |
| 바뀐 내용 | “너는 누구야?에 holiday로 답하는 절 추가” | 에이전트·사용자가 이해 |
| 손댔을 때 | “절만 추가하고 사용자 문장은 유지” | 수정분 보존 방법 |

**2) 장부 폴더 — 지금까지 맞춰 둔 위치 (지문)**

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 마지막 맞춤 버전 | `0.4.1` | 어디부터 쪽지를 더 읽어야 하는지 |
| 반영한 항목 | `presence-intro` 반영됨 | 같은 항목을 매 세션 반복하지 않음 |
| 파일 지문 | `AGENTS.md` 해시 | 복사 뒤 사용자가 고쳤는지 |

지문은 장부 워크스페이스 안(예: `.holiday/` 아래)에 둔다.

## 목표

1. 복제본에 영향을 주는 릴리스마다 플러그인에 **변경 쪽지**를 남긴다.
2. 쪽지 항목은 **필수**와 **선택**으로 나눈다.
3. 에이전트가 미반영 쪽지를 읽고 사용자에게 보고한 다음 맞춘다.
   - 필수: 빠뜨리지 않고 알리고 반영·보류(이유 포함) 중 결정을 받는다
   - 선택: 짧게 제안한다. 매 메시지마다 반복하지 않는다
4. 대상 파일이 사용자가 손댄 상태면 쪽지의 “손댔을 때” 안내를 따르고
   확인 없이 통째로 덮지 않는다.
5. 문서 스킬(`skills update`)·세션 시작 훅은 복제본을 **쓰지 않는다**.
6. 원장 DB, dash 메모·조종석은 기본 맞춤 대상이 아니다.

## Non-goals

- `skills update`에 장부 지침 갱신을 섞지 않는다
- 세션 시작만으로 관리 파일을 자동 쓰지 않는다
- 필수 항목을 사용자 몰래 반영하지 않는다 (알림·결정 없는 강제 적용 금지)
- 선택 항목을 필수처럼 매 세션 재촉하지 않는다
- 세 방향 병합 전용 UI를 만들지 않는다
- 정체성·말투 **문구 내용**은 별도 기획에 맡긴다. 이 PRD는 **쪽지·전달·맞춤 경로**만 다룬다

## 비기능

- 쪽지는 플러그인에만 둔다. progressive disclosure로 필요할 때 읽는다
  (SKILL.md에는 “쪽지를 읽어라” 정도만).
- 장부 지문이 없어도 대화는 막지 않는다. “맞춤 이력 없음”으로 보고하고
  보수적으로 진행한다.
- CLI는 지문·상태 확인을 도울 수 있으나 **본선 UX는 에이전트 + 플러그인 쪽지**다.

## 성공 지표

- 새 릴리스에 복제본 변경이 있으면 플러그인 쪽지에 필수/선택이 보인다
- 에이전트가 미반영 필수를 사용자에게 빠짐없이 알린다
- 선택은 한 번 제안하고 거절·보류 후 같은 세션에서 반복 재촉하지 않는다
- 손댄 파일은 통째 덮기 전에 안내·확인이 있다
- 세션 시작만으로 `AGENTS.md`·dash 사용자 파일이 바뀌지 않는다
