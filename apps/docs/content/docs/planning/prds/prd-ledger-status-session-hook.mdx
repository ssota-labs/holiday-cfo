## 한 줄

장부 폴더에서 세션이 열릴 때 **현황 파일이 있는지 확인**한다. 없으면 장부
기준으로 한 장을 만들어 둔다.

## 문제

[prd-009](/docs/planning/prds/prd-ledger-status-md)로 현황 파일·다시 쓰기·에이전트
안내는 갖춰졌다. 그래도 빈틈이 남는다.

- 에이전트가 안내를 잊고 목록 조회부터 돌린다
- 예전에 만든 장부 폴더의 안내 파일은 새 문구로 바뀌지 않는다
- 현황 파일이 없으면 세션마다 먼저 만들라는 말을 반복해야 한다

사람이 매번 기억할 일이 아니다. 세션이 열릴 때 **파일 유무만** 확인해, 없을 때만
만들면 된다.

## 무엇을 저장·보여 주는가

장부 DB에 새 칸은 없다. 세션 시작 때 일어나는 일이고, 필요할 때만 생기는 파일이다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 현황 파일 유무 | `status.md` 있음/없음 | 세션 초반에 구조를 읽을 파일이 있는지 본다 |
| 없을 때 만들기 | 없는 폴더에서 세션을 열면 파일이 생긴다 | 에이전트가 안내를 잊어도 빈손이 아니다 |
| 이미 있을 때 | 있는 파일은 그대로 둔다 | 매 세션마다 통째로 덮지 않는다 |
| 실패해도 대화 | 네트워크·CLI 실패여도 채팅은 이어진다 | 훅이 장부 대화를 막지 않는다 |

만드는 본문은 [prd-009](/docs/planning/prds/prd-ledger-status-md)와 같다.
계좌·부채 이름만, 잔액은 없다.

## 목표

1. 유저 장부 폴더에서 세션이 열리면 현황 파일 유무를 확인한다
2. 없으면 기존 다시 쓰기와 같은 내용으로 파일을 만든다
3. 이미 있으면 손대지 않는다 (오래됨·손편집은 에이전트·다시 쓰기 명령이 맡는다)
4. 확인·만들기가 실패해도 대화를 막지 않는다

## Non-goals

- 파일이 있을 때마다 세션마다 다시 쓰기
- 잔액·현금흐름·마감 숫자를 훅이 검사하거나 파일에 넣기
- 현황 파일 말고 임의 파일 목록 전체를 세션마다 보고하기
- 예전에 복사된 안내 파일·훅 설정을 자동으로 덮어 고치기
- 장부 없는 폴더·개발용 모노레포에서 파일을 만들기

## 비기능

- 대상은 `.holiday/`가 있는 **유저 장부 프로젝트**다. holiday 개발 모노레포는
  건너뛴다
- Cursor는 프로젝트 훅, Claude Code·Codex는 플러그인 세션 훅을 쓴다. 호스트가
  훅을 거절하거나 신뢰 전이면 안내·수동 다시 쓰기로 같은 결과를 낼 수 있어야 한다
- 이미 있는 프로젝트 훅 설정은 init이 덮지 않는다. 새 템플릿은 새 장부에만
  기본으로 들어가고, 기존 장부는 사람이 훅을 한 줄 더하거나 스크립트를 추가한다

## 성공 지표

1. 장부 폴더에 현황 파일이 없을 때 세션을 열면 파일이 생긴다 (훅이 켜진 호스트)
2. 이미 파일이 있으면 세션을 열어도 내용이 바뀌지 않는다
3. 훅이 실패해도 채팅은 계속된다
4. 장부 없는 폴더·개발 모노레포에서는 현황 파일을 만들지 않는다
5. 에이전트·기능 문서에 “세션 시작 때 유무 확인, 없으면 만들기”가 적혀 있다
