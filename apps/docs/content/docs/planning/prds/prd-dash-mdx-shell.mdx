## 한 줄

장부 옆 화면 폴더는 **문서 사이트와 같은 fumadocs 앱**으로 둔다. 사용자는
메모·기록을 마크다운 페이지로 쓰고, 금액·잔액·현금 여유는 **별 대시보드
페이지**에서 장부 API로만 본다.

## 문제

지금 장부 화면은 가벼운 단일 페이지 앱이다. 배치 파일 한 장으로 블록만 고른다.
잘못된 금액을 넣지는 못하지만 아래가 불편하다.

- 여러 페이지(개요 / 카드 / 월간 메모)로 나누기 어렵다
- 문장과 숫자 칸을 한 흐름으로 쓰기 어렵다
- 배치가 JSON이라 사람이 읽고 고치기 어렵다
- 메모·결정·회고를 남길 자리가 한 줄 메모 칸 수준이다

문서 사이트는 이미 fumadocs로 페이지·테마·커스텀 칸을 잘 운영한다. 같은 방식을
**사용자 각자의 장부 프로젝트 폴더**에 깔면, 메모는 MDX로 쓰고 숫자 화면은
대시보드 라우트로 나눌 수 있다. holiday 모노레포의 문서 사이트와 합치는
일이 아니다.

## 무엇을 저장·보여 주는가

장부 DB 스키마는 바꾸지 않는다. 아래 항목은 사용자 프로젝트의 화면 폴더에 남는다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 화면 앱 | 장부 옆 `dash/` (fumadocs) | 로컬에서 보고, 배포 시 읽기 전용 사진 |
| 메모·기록 페이지 | `content/journal/2026-07.mdx` | 회고·결정·맥락 |
| 숫자 대시보드 | `/dashboard` 같은 별 페이지 | 잔액·런웨이·분류 대기 |
| 장부 스냅샷/응답 | API가 돌려주는 잔액·여유·건강 | 숫자는 장부에서만 |
| 허용 칸 | 현금 여유표, 잔액표, UI 카드 등 | 사람이 금액을 직접 못 넣게 |

## 목표

1. `dash init`이 사용자 폴더에 **fumadocs 기본 컴포넌트·테마**를 쓰는 앱을 깐다.
2. 기존 화면용 UI(셸·칸)와 가계부 블록을 **같은 앱**에서 쓰고, MDX에도
   전역 등록해 태그로 넣는다.
3. 메모·중요 기록은 MDX 페이지로 쓴다. 숫자 대시보드는 **별 라우트**다.
4. 금액·잔액·런웨이·부족 구간은 **API(로컬에서는 장부 CLI 경유)** 로만 읽는다.
   페이지 본문에 금액을 직접 적어 “현재 잔액”처럼 보이게 하지 않는다.
5. 예전의 JSON 배치 렌더러는 제거한다. 배치는 MDX·대시보드 페이지가 맡는다.
6. 로컬에서는 분류 대기를 클릭으로 처리한다. 배포본은 읽기 전용 사진이다.
7. 웹 화면은 보조다. 장부 본체는 여전히 CLI와 채팅 shell-out이다.

## Non-goals

- holiday 모노레포 문서 사이트(`apps/docs`)와 장부 화면을 한 앱으로 합치기
- 브라우저가 장부 DB 파일을 직접 열기
- 문서 전용 칸(정책 규칙, 스키마 표 등)을 장부 화면 어휘에 그대로 섞기
- 금액 prop·쓰기 액션을 페이지 작성 어휘에 노출하기
- 카드사·세액·할부 공식을 화면이 다시 계산하기
- 멀티유저 클라우드 가계부 제품으로 키우기

## 비기능

- 스냅샷·API 금액은 문자열(소수 표기)로 끝까지 유지한다. 화면이 일반 숫자 타입으로
  왕복하지 않는다.
- 한 시점에 읽는 잔액·현금흐름·검증은 서로 어긋나면 안 된다.
- 현금 여유의 **부족 구간**은 숨기지 않는다.
- 화면 패키지 버전은 CLI와 같은 태그로 맞춘다.
- 로컬 조종석 / 배포 사진 경계는 유지한다.

## 성공 지표

- 사용자는 fumadocs 테마의 화면 앱에서 메모 페이지와 숫자 대시보드를 함께 본다
- MDX에 허용된 UI 칸·가계부 블록을 태그로 넣을 수 있다
- 페이지에 금액을 직접 적어도, 화면에 권위 있는 숫자로 나타나지 않는다
- 장부를 바꾼 뒤 API/스냅샷을 갱신하면 페이지 글은 그대로이고 숫자만 바뀐다
- 로컬에서 분류 대기 한 건을 클릭으로 처리한다
- 배포된 화면에는 쓰기 UI가 동작하지 않는다
- JSON 배치 파일 없이도 화면이 구성된다

## 결과 계약

장부 화면은 **fumadocs 앱 + 스냅샷/API 뷰어**다. MDX는 기록과 배치를 맡고,
숫자는 API·스냅샷만 맡는다. 기계 계약은
[SPEC-dash-mdx-shell](/docs/spec/dash-mdx-shell)과
[ADR-012](/docs/development/adr/adr-012)에 둔다.
