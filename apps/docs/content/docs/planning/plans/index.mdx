구현계획은 PRD와 스펙을 패키지·파일·테스트 단위 실행 순서로 바꾼다. 제품 계약이 아니라
특정 변경을 완료하기 위한 기간성 문서다.

| ID | Stage | Change type | PRD |
|---|---|---|---|
| [PLAN-docs-first-development](/docs/planning/plans/plan-docs-first-development) | done | product | PRD-docs-first-development |
| [PLAN-human-friendly-ledger-voice](/docs/planning/plans/plan-human-friendly-ledger-voice) | done | product | PRD-human-friendly-ledger-voice |

## 상태

- `draft`: 검토 전
- `ready`: 구현을 시작할 수 있음
- `active`: 구현 중
- `done`: 구현과 검증 완료
- `superseded`: 다른 계획으로 대체

구현 PR은 base 브랜치에 `ready` 또는 `active` 상태로 존재하는 계획만 참조할 수 있다.
