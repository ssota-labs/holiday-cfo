**As a** 회계와 개발에 익숙하지 않은 사용자  
**I want** 장부 작업 결과를 평소 쓰는 말로 정확하게 설명받고 싶다  
**So that** 계정 코드와 전표 구조를 해석하지 않고도 내 돈이 어떻게 달라졌는지 알 수 있다

### 수락 기준

1. 답변은 사용자가 요청한 결과를 첫 문장에 말한다.
2. 명령어·파일·계정 코드·전표 구조는 사용자가 요청하거나 확인에 꼭 필요할 때만 보인다.
3. 회계 용어가 필요하면 일상어 설명을 같은 문장에 붙인다.
4. 두세 문장으로 충분한 내용을 제목 여러 개나 표로 늘리지 않는다.
5. 숫자의 값은 바꾸지 않고 `800,481원`처럼 단위와 구분을 명확히 표시한다.
6. 전송 전에 맞춤법·띄어쓰기·조사·숫자·단위를 확인한다.
7. 질문은 결과를 바꾸는 선택이 있을 때만 한 번에 하나씩 한다.
