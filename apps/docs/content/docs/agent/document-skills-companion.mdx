## 범위

장부 플러그인(`holiday-cfo`)이 **문서 형식 스킬을 companion으로 붙이는** 계약이다.
원장 스키마·CLI 플래그는 바꾸지 않는다.

**대상 cwd:** 플러그인을 쓰는 **유저별 장부 프로젝트** (보통 `holiday init`으로
만든 `.holiday/`가 있는 디렉터리). holiday-cfo 모노레포·개발자 pin은 범위 밖이다.

대상 스킬(Anthropic `anthropics/skills`):

| 스킬 | 역할 |
|---|---|
| `xlsx` | 스프레드시트 |
| `pdf` | PDF |
| `docx` | 워드 |
| `pptx` | 슬라이드 |

**기본 경로:** Cursor·Codex·Claude Code 모두 `npx skills` **프로젝트 설치**.  
`-g`(전역)와 Anthropic 마켓 `document-skills@…`는 기본 경로가 아니다.

## 인터페이스

### 1) 첫 설치 안내 (셋업 워크플로)

`plugins/*/skills/holiday-cfo/references/workflows/setup.md` (및 SKILL.md에서
셋업으로 연결되는 문구)에 companion 설치를 넣는다.

에이전트는 **장부 프로젝트 루트**(세션 cwd / `${CLAUDE_PROJECT_DIR}` 등)에서
아래를 안내·실행한다.

**전 호스트 공통 — skills.sh 프로젝트 설치 (기본, 유일한 정본)**

```bash
npx skills add https://github.com/anthropics/skills --skill xlsx -y
npx skills add https://github.com/anthropics/skills --skill pdf -y
npx skills add https://github.com/anthropics/skills --skill docx -y
npx skills add https://github.com/anthropics/skills --skill pptx -y
```

`-g`를 붙이지 않는다. 결과는 그 프로젝트에:

| 경로 | 역할 |
|---|---|
| `<project>/.agents/skills/<name>/` | 스킬 본문 |
| `<project>/skills-lock.json` | 프로젝트 잠금(해시) |

에이전트 타깃을 고를 수 있으면 현재 호스트를 포함해 링크한다
(예: `-a claude-code`). discovery 경로는 skills CLI 버전에 따르되, **명령 골격은
위와 같다**.

### 2) 세션 시작 갱신

이미 **그 프로젝트**에 설치된 스킬만 대상으로 한다.

```bash
npx skills update -p -y
```

| 항목 | 계약 |
|---|---|
| cwd | 유저 장부 프로젝트 루트 |
| 범위 | 프로젝트만 (`-p`). 전역 `~/.agents/.skill-lock.json`은 수정하지 않음 |
| 잠금 | `<project>/skills-lock.json` 해시·시각 갱신 |
| 실패 | exit non-zero여도 세션·대화를 차단하지 않음 (soft-fail) |
| 타임아웃 | 호스트가 허용하는 합리적 상한(권장 ≤ 120s) |
| 미설치 | 프로젝트 잠금/스킬이 없으면 no-op. 첫 설치 안내를 대체하지 않음 |

> 잠금 파일은 백그라운드 데몬이 아니다. **`skills update -p`가 돌 때만** 그
> 프로젝트 잠금이 최신 해시로 갱신된다.

### 3) 호스트별 훅 배치

| 호스트 | 배치 | 명령 |
|---|---|---|
| Claude Code | `plugins/claude-code/hooks/hooks.json` → `SessionStart` | 프로젝트 cwd에서 `npx skills update -p -y` (soft-fail) |
| Codex | `plugins/codex/hooks/hooks.json` → `SessionStart` | 동일. 플러그인 훅 **trust** 필요할 수 있음 |
| Cursor | `holiday init`이 쓰는 프로젝트 `.cursor/hooks.json`의 `sessionStart` | 동일 |

공유 스크립트를 둘 경우 플러그인 루트 또는 CLI 템플릿 한곳만 소스로 삼고,
호스트별 JSON은 그 스크립트를 가리킨다. Anthropic 스킬 **본문은 복사하지 않는다.**

훅은 **현재 워크스페이스가 유저 장부 프로젝트일 때** 의미가 있다. holiday-cfo
모노레포 개발 세션을 대상으로 하지 않는다(구현 시 cwd/마커로 가드 가능 —
예: `.holiday/` 존재 시에만 update).

## 동작

1. 셋업 중 에이전트는 **이 장부 프로젝트**에 문서 스킬이 있는지 확인한다.
   없으면 위 `npx skills add … -y`를 import보다 먼저 안내·실행한다.
2. 이후 그 프로젝트 세션: 훅이 `npx skills update -p -y`를 실행한다.
3. 원격 해시가 같으면 다운로드 없음. 다르면 해당 스킬만 재설치하고 프로젝트
   잠금을 갱신.
4. 훅 실패·타임아웃·미설치는 무시하고 장부 스킬 흐름을 계속한다.

## 데이터와 불변식

| 사람이 읽는 이름 | 코드/경로 | 예시 |
|---|---|---|
| 유저 프로젝트 스킬 | `<ledger>/.agents/skills/xlsx/` 등 | 프로젝트 설치 결과 |
| 유저 프로젝트 잠금 | `<ledger>/skills-lock.json` | skills.sh project lock |
| holiday 모노레포 pin | (개발 저장소) `skills-lock.json` | 이 계약 범위 밖 · CI `check:skills` |

불변식:

1. `plugins/*/skills/` 아래에 `xlsx`/`pdf`/`docx`/`pptx` 벤더 트리를 두지 않는다.
2. 기본 설치·갱신에 `-g`를 쓰지 않는다.
3. ADR-009와 같이 upstream 본문은 wrap·load만 하고 paste하지 않는다.
4. 셋업·README에 Claude `/plugin install document-skills@…`를 기본으로 적지 않는다.

## 오류

| 상황 | 기대 |
|---|---|
| `npx`/네트워크 실패 | log 기록(옵션), 세션 계속 |
| 프로젝트 스킬 없음 | no-op |
| cwd가 장부 프로젝트가 아님 | update 스킵(가드) 또는 no-op |
| Codex 훅 untrusted | 훅 스킵. 셋업 안내로 수동 `skills update -p` 가능해야 함 |
| Cursor가 `sessionStart`를 거절 | init/문서에 대체 등록법(또는 수동 update)을 남김 |

## 호환성

- CLI·원장 스키마·exit code 계약 불변
- 유저 프로젝트의 `.agents/skills/`·`skills-lock.json`은 사용자 저장소에 속한다
  (원장 DB와 별개). gitignore 여부는 사용자 선택 — 기본 강제하지 않음
- 플러그인 버전 bump는 훅·셋업 문구가 사용자-facing이면 릴리스 게이트 적용
- 기존 장부 디렉터리: init 재실행 시 Cursor 훅 파일이 없을 때만 생성

## 수락 기준

1. setup에 skills.sh **프로젝트** 설치(`-y`, `-g` 없음) 네 스킬이 정본으로 있다
2. setup에 전역 `-g`나 Claude marketplace `document-skills`를 기본으로 쓰지 않는다
3. Claude·Codex 플러그인에 soft-fail `SessionStart` → `npx skills update -p -y`가 있다
4. Cursor용 `sessionStart`가 init 템플릿 또는 동등 문서로 제공된다
5. 플러그인 tree에 Anthropic document skill 본문이 없다
6. 계약·구현이 holiday 모노레포 개발용 pin을 대상으로 하지 않는다
