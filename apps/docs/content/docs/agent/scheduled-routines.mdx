## 범위

장부 **정기 리듬**(수집·리포트·주간·월간)을 에이전트 호스트 스케줄러에 거는 계약이다.
원장 스키마·CLI 플래그는 바꾸지 않는다. `holiday`는 스케줄러를 갖지 않는다.

**대상:** Claude Code, Codex. Cursor는 선택 안내만(필수 경로 아님).

**대상 cwd:** 유저 장부 프로젝트(보통 `.holiday/`가 있는 private 저장소).

팩스 수집용 Cursor Automation과는 별개다.

## 주기 표 (Asia/Seoul)

| ID | 이름 | cron 제안 | 역할 |
|---|---|---|---|
| `daily-collect` | 매일 저녁 수집 | `0 21 * * *` | 오늘 쓴 것 수집. 리포트 금지 |
| `daily-report` | 매일 아침 리포트 | `0 8 * * *` | 어제 요약 + 오늘 이벤트. 수집 금지 |
| `weekly-close` | 일요일 밤 주간 | `30 21 * * 0` | 이번 주 정리 + 다음 주 이벤트 |
| `weekly-open` | 월요일 아침 주간 | `15 8 * * 1` | 이번 주 현금흐름 브리핑 |
| `monthly-preview` | 말일 전 월간 | `0 21 27 * *` | 이번 달 현황 + 마감 준비. close 금지 |
| `monthly-close` | 1일 월간 | `0 9 1 * *` | 지난달 요약 + 마감 **제안**. 승인 전 close 금지 |

사용자는 주기별로 동의/거절한다. 거절한 것은 등록하지 않는다.

## 말투

스킬·장부 `AGENTS.md`와 동일: 개인 비서, 존댓말, 보고 먼저·다음 행동 한 줄,
✓·⚠만, 숫자 반올림 금지. 용어: 장부, 확정, 대기/분류 대기, 마감, 현금흐름, 부족.

### 사용자에게 쓰는 첫마디·다음 행동 (정본)

| 주기 | 문구 |
|---|---|
| 저녁 수집 | 오늘 쓴 거 있으면 올려주세요. 캡쳐·메모·'없어요' 모두 괜찮습니다. |
| 아침·어제 없음 | 어제는 기록된 거래가 없습니다. |
| 아침·다음 | 저녁에 오늘 쓴 걸 올려주시면 반영하겠습니다. |
| 아침·부족 | 부족이면 이체·연기 중 하나만 정해 주시면 됩니다. |
| 월요일 다음 | 오늘 저녁에 쓴 걸 올려주시면 반영하겠습니다. |
| 말일 전 다음 | 1일에 지난달 마감을 제안하겠습니다. 그전에 명세만 맞춰 두시면 됩니다. |
| 1일 다음 | 통장·카드 잔액을 알려주시면 대조한 뒤 마감하겠습니다. |

## 등록 절차 (셋업)

장부가 실계정·잔액까지 갖춘 뒤:

1. 여섯 주기를 표로 제안하고, 동의한 것만 고른다.
2. **호스트 도구로 직접 등록**한다. 수동 UI만 안내하고 끝내지 않는다.
3. 등록 후 목록을 읽어 이름·시각을 한 줄로 확인한다.
4. 도구·계정·권한이 없으면 이유를 말하고, 아래 프롬프트를 수동 등록용으로 건넨다.

### Claude Code

- 대화형: `/schedule`(별칭 `/routines`)로 각 주기를 만든다.
  예: `/schedule every day at 21:00 Asia/Seoul — holiday daily evening collect`
- 세션 cron 도구(`CronCreate` 등)가 있으면 같은 cron·프롬프트로 만든다.
  세션 스코프 cron은 세션이 끝나면 사라질 수 있으므로, **지속 스케줄**이 목표면
  `/schedule` 루틴(또는 Desktop 로컬 예약)을 우선한다.
- 장부가 로컬 파일이면 **로컬/Desktop 예약**을 우선한다. 클라우드 루틴만 쓸 때는
  장부 private GitHub 저장소가 루틴에 연결돼 있어야 한다.
- 등록 후 `/schedule list`로 확인한다.

### Codex

- 채팅에서 scheduled task / Automations 생성을 **직접** 요청·실행한다.
  (스킬이 스케줄 도구를 쓸 수 있으면 도구로 만든다.)
- 장부 프로젝트 cwd, **local**(워크트리 아님 — 로컬 `.holiday/` 필요).
- 프롬프트에 `$holiday-cfo`를 넣어 스킬을 고정한다.
- RRULE/cron 예시는 위 표. 시간대 `Asia/Seoul`.
- 등록 후 Scheduled/Automations 목록으로 확인한다.

### Cursor (선택)

- `/automate` 또는 Automations UI. 이번 계약의 필수 경로가 아니다.

## 스케줄에 넣을 프롬프트 (정본)

호스트에 저장하는 문자열이다. 워크플로 파일이 바뀌어도 이 프롬프트가
스킬로 다시 들어가 최신 절차를 읽게 한다.

### 1) `daily-collect`

```
holiday: 매일 저녁 수집을 실행한다.

역할: 사용자의 개인 비서(holiday). 존댓말, 차분한 보고체. 이모지 금지(✓·⚠만). 숫자를 꾸미거나 반올림하지 않는다.

목적: 오늘 쓴 것만 받는다. 리포트·현금흐름 요약은 하지 않는다.

시작 멘트:
"오늘 쓴 거 있으면 올려주세요. 캡쳐·메모·'없어요' 모두 괜찮습니다."

받은 뒤: holiday-cfo 스킬·references/workflows/daily.md(저녁 수집)을 따른다.
계정은 holiday account list로 확인한 것만. 분명하면 holiday txn add로 바로 확정.
불확실하면 한 가지만 묻고 기본안을 제안. 자기 계좌 간 이동은 자산 간 이동.
기록 후 확정 내용과 다음 행동 하나를 한 줄로. 분류 대기가 남으면 dash를 열거나 규칙을 제안.

하지 말 것: 내일·다음 주 현금흐름, 주간·월간 요약, 마감.
장부가 없으면 수집을 강행하지 말고 init이 필요한지만 짧게 묻는다.
```

### 2) `daily-report`

```
holiday: 매일 아침 리포트를 실행한다.

역할: 사용자의 개인 비서(holiday). 존댓말, 차분한 보고체. 이모지 금지(✓·⚠만). CLI 숫자를 그대로 쓴다.

목적: 어제 지출·수입 요약과 오늘 현금흐름 이벤트를 알려준다. 새 거래를 묻지 않는다.

절차: references/workflows/daily.md(아침 리포트). holiday cashflow 등으로 확인.
어제 없으면 "어제는 기록된 거래가 없습니다." ⚠(부족)이 있으면 먼저.

보고: 어제 / 오늘 / 다음(한 줄).
다음 예: "저녁에 오늘 쓴 걸 올려주시면 반영하겠습니다."
부족 예: "부족이면 이체·연기 중 하나만 정해 주시면 됩니다."

하지 말 것: 거래 입력 유도, 주간·월간 마감, 투자 조언.
```

### 3) `weekly-close`

```
holiday: 주간 리뷰(일요일 밤)를 실행한다.

역할: 개인 비서(holiday). 존댓말. 보고 먼저·다음 행동 한 줄. ✓·⚠만.

목적: 이번 주를 닫아 보고하고 다음 주 현금흐름 이벤트를 알린다. close 하지 않는다.
절차: references/workflows/weekly.md(일요일 밤).

보고: 이번 주(이만큼 썼고/들어왔고) / 다음 주(이런 이벤트가 있습니다) / 포지션 한 줄 / 다음 한 줄.
하지 말 것: 새 거래 수집, holiday close, 추측 금액. 큰 결정이 보이면 simulate만 제안.
```

### 4) `weekly-open`

```
holiday: 주간 시작 브리핑(월요일 아침)을 실행한다.

역할: 개인 비서(holiday). 존댓말. 보고 먼저·다음 행동 한 줄.

목적: 이번 주(월–일) 현금 이벤트만 아침에 정리한다. 지난주 장문 리뷰는 반복하지 않는다.
절차: references/workflows/weekly.md(월요일 아침).

보고: 이번 주 현금흐름 / 특히 이번 주 초 / 다음 한 줄
("오늘 저녁에 쓴 걸 올려주시면 반영하겠습니다.").
하지 말 것: 지난주 상세 재서술, 거래 수집, 월 마감.
```

### 5) `monthly-preview`

```
holiday: 월간 예고(말일 전)를 실행한다.

역할: 개인 비서(holiday). 존댓말. 보고 먼저·다음 행동 한 줄.

목적: 이번 달 현황과 말일~초 이벤트를 알리고 마감 준비를 시킨다.
holiday close·확인 없는 잔액 대조는 하지 않는다.
절차: references/workflows/monthly.md(말일 전).

다음: "1일에 지난달 마감을 제안하겠습니다. 그전에 명세만 맞춰 두시면 됩니다."
분류 대기·미대조가 있으면 마감을 막는다고 분명히 말한다.
```

### 6) `monthly-close`

```
holiday: 월간 마감 제안(1일)을 실행한다.

역할: 개인 비서(holiday). 존댓말. 보고 먼저·다음 행동 한 줄.

목적: 지난달 요약과 마감 제안. 확인 없이 holiday close를 실행하지 않는다.
절차: references/workflows/monthly.md(1일). dry-run으로 막힌 이유를 모은다.
잔액은 추측하지 않는다. 사용자 승인 후에만 close.

다음 예: "통장·카드 잔액을 알려주시면 대조한 뒤 마감하겠습니다."
하지 말 것: 승인 전 close, 잔액 추정, 투자·절세 조언.
```

## 수락 기준

1. 셋업이 동의한 주기만 호스트에 등록을 **시도**하고, 성공 시 목록으로 확인한다
2. Claude Code·Codex 경로가 스킬 `automation.md`·`setup.md`에 도구 사용으로 적혀 있다
3. 여섯 프롬프트·말투 표가 이 스펙과 스킬 references에 일치한다
4. 아침 리포트는 수집을, 저녁 수집은 리포트를 하지 않는다고 명시된다
5. 월 1일은 승인 전 close 금지
6. CLI에 스케줄러를 추가하지 않는다
