## 범위

유저 장부 프로젝트에 **현황 마크다운**을 두고, 계좌·부채 이름만 나열한다.
원장 스키마를 바꾸지 않는다. 잔액·현금흐름·수익·비용 차트 전체 나열은 1차
범위 밖이다.

**대상 cwd:** `.holiday/ledger.db`가 있는 장부 프로젝트.

세션 시작 훅은 같은 cwd에서 **파일 유무만** 보고, 없을 때만 다시 쓰기와 동일
본문으로 만든다. 문서 스킬 companion 훅과 같은 soft-fail 계열이다.

## 사람 말로

지금 어떤 통장·카드·대출·할부가 있는지 파일 하나로 본다. 숫자는 믿지 말고
이름만 본다. 파일이 없거나 오래돼 보이면 다시 쓰거나 목록 조회로 확인한다.
장부 폴더에서 채팅이 열리면, 파일이 없을 때는 훅이 먼저 만들어 둘 수 있다.

## 인터페이스

| 표면 | 동작 |
|---|---|
| 현황 파일 | 경로: 장부 프로젝트 루트의 `status.md` (UTF-8 마크다운) |
| 다시 쓰기 명령 | `holiday status`. 장부를 읽어 파일을 **통째로** 덮어쓴다. 손편집 보존 없음 |
| 세션 시작 훅 | `.holiday/` 장부 cwd에서 `status.md` 유무 확인. **없으면** 다시 쓰기와 동일 본문으로 생성. **있으면** no-op |
| 구조 변경 후 | 계정·카드·대출·할부 **추가·개명·종료**에 성공하면 다시 쓰기를 권장한다. 구현은 자동 다시 쓰기를 택할 수 있다 (명령과 동일 본문) |
| 에이전트·스킬 | 세션·구조 파악에서 `status.md`를 **먼저** 확인한다. 없거나 비면 다시 쓰기 또는 목록 조회로 폴백한다 |
| stdout | 다시 쓰기 명령은 사람용 한 줄(`note`) + `--json` 시 경로·갱신시각 |

변경 명령의 플래그·exit code 세부는 CLI 스펙 페이지에서 맞춘다 (구현 PR).

## 동작

### 파일 본문 (필수 절)

다시 쓴 결과는 최소한 다음 절을 이 순서로 가진다.

1. **제목** — 고정: `# 장부 현황`
2. **갱신 시각** — 타임존 포함 ISO-8601 (예: `2026-07-22T17:00:00+09:00`)
3. **계좌** — 자산 계정 중 **자리표시(placeholder)가 아닌** 잎 계정.
   쓸 수 있는 돈(현금 플래그)이면 표지에 표시한다
4. **부채** — 하위 절:
   - 카드 (청구주기가 붙은 카드 부채)
   - 대출
   - 할부
   - 그 외 부채 계정 (미지급 등, 있을 때만)

각 행은 **사람 이름(또는 계정 코드의 마지막 구간) + 계정 코드**를 함께 쓴다.
코드는 에이전트가 그대로 쓰기 위한 앵커다.

### 넣지 않는 것 (1차)

- 잔액·순자산·유동/비유동 합계
- 수익·비용·자본 계정 목록
- 정기지출·정기수입 스케줄 (별도 목록 명령이 있음)
- 사용자가 끼워 넣은 자유 메모 (다시 쓰기 시 삭제)

### 다시 쓰기 규칙

1. 원장이 없으면 명령을 거절한다 (기존 원장 없음 오류와 동일 계열)
2. 대상 파일이 없어도 만든다. 있으면 전체를 덮는다
3. 같은 원장 상태에서 두 번 쓰면, 갱신 시각만 다를 수 있고 목록 절은 동등하다
4. 파일을 손편집한 뒤 다시 쓰면 손편집은 사라진다 — 의도된 동작이다

### 에이전트 읽기 규칙

1. 장부 폴더에서 일할 때 `status.md`를 **항상** 먼저 확인한다
2. 파일이 없거나 절이 비어 있으면 `holiday status`를 돌리거나 `account list` /
   `card list` / `loan list` / `installment list`로 폴백한다
3. **잔액·확정·대조**에는 이 파일을 쓰지 않는다 — 기존 조회·대조 명령을 쓴다
4. 파일에 없는 계정 이름을 지어내지 않는다 (기존 “계정 추측 금지”와 동일)

### 세션 시작 훅 규칙

1. **가드:** `.holiday/`가 있는 유저 장부 프로젝트에서만 동작. holiday 개발
   모노레포·장부 없는 cwd는 no-op
2. **없을 때만 생성:** `status.md`가 없으면 `holiday status`와 동일 본문으로
   만든다. 있으면 내용을 덮지 않는다
3. **soft-fail:** CLI·네트워크·권한·호스트 trust 실패는 세션을 막지 않는다
   (exit 0 계열, 문서 스킬 companion와 동일 정신)
4. **호스트 배치:**
   - Claude Code / Codex: 플러그인 `SessionStart` → 플러그인 루트 스크립트
   - Cursor: `holiday init` 템플릿 `.cursor/hooks.json` `sessionStart` (없는
     파일만 생성). 이미 `hooks.json`이 있는 장부는 수동으로 항목·스크립트를
     더한다
5. **컨텍스트(선택):** 호스트가 허용하면 “생성함 / 이미 있음 / 건너뜀” 한 줄.
   잔액·계정 목록 전체를 훅 출력에 넣지 않는다

## 데이터와 불변식

| 사람이 읽는 이름 | 코드 키(안내용) | 예시 |
|---|---|---|
| 파일 경로 | `statusPath` | `status.md` (cwd 기준) |
| 갱신 시각 | `generatedAt` | `2026-07-22T17:00:00+09:00` |
| 계좌 행 | `assets[]` | 코드 + 현금 여부 |
| 카드 행 | `cards[]` | 부채 계정 코드 + 청구 요약(선택) |
| 대출 행 | `loans[]` | 부채 계정 코드 + 라벨 |
| 할부 행 | `installments[]` | 할부 부채 계정 + 라벨 |

불변식:

1. SoR는 `.holiday/ledger.db`다. `status.md`만으로 계정·스케줄을 만들지 않는다
2. 다시 쓰기 직후, 파일의 계정 코드 집합은 장부의 해당 목록과 일치한다
3. 1차 본문에 금액(minor unit) 칸을 넣지 않는다

## 오류

| 상황 | 결과 |
|---|---|
| 원장 없음 | 거절, 파일 미작성 |
| 쓰기 권한 없음 | 거절, 기존 파일 유지(있으면) |
| 손편집만 있고 원장과 불일치 | 오류 아님 — 다음 다시 쓰기가 원장으로 맞춤 |
| 세션 훅에서 CLI/네트워크 실패 | soft-fail, 세션 계속. 에이전트는 수동 다시 쓰기·목록 폴백 |
| Cursor/Codex 훅 거절·untrusted | 훅 스킵. 안내·수동 `holiday status`로 동일 결과 가능해야 함 |

## 호환성

- 기존 장부에 `status.md`가 없어도 동작한다 (없으면 읽고 폴백, 훅이 있으면 생성)
- 마이그레이션 없음
- dash `data/ledger.json`·MDX와 경로·역할이 겹치지 않는다
- 기존 장부의 `.cursor/hooks.json`·플러그인 밖 설정은 init이 덮지 않는다

## 기존 Cursor 장부 — 수동 병합

`holiday init`은 **없는 파일만** 쓴다. 이미 `.cursor/hooks.json`이 있는 장부에는
ensure 훅이 자동으로 안 붙는다. 아래를 한 번 더한다.

1. 스크립트 복사 (없으면): 최신 CLI 템플릿의
   `.cursor/hooks/ensure-status-md.sh`를 장부
   `.cursor/hooks/ensure-status-md.sh`로 둔다 (실행 권한 `+x`).
2. `.cursor/hooks.json`의 `hooks.sessionStart` 배열에 항목을 **추가**한다
   (기존 문서 스킬 훅은 유지):

```json
{
  "command": "./.cursor/hooks/ensure-status-md.sh",
  "timeout": 120
}
```

3. 훅을 거절했거나 CLI가 없으면 세션은 그대로 이어지고, 수동으로
   `holiday status`(또는 `npx @holiday-cfo/cli@latest status`)를 돌리면 같은
   파일을 만든다.

Claude Code·Codex는 플러그인 갱신으로 `SessionStart`에 ensure가 들어간다.
trust가 꺼져 있으면 위 수동 다시 쓰기와 같다.

## 수락 기준

1. 다시 쓰기 후 `status.md`에 제목·갱신 시각·계좌·부채 절이 있다
2. 장부에 있는 비-placeholder 자산 잎 계정과 카드·대출·할부가 각 절에 나타난다
3. 본문에 잔액 숫자가 없다
4. 파일만 수정해도 원장 계정·스케줄이 변하지 않는다
5. 스킬/장부 안내에 “세션·구조 파악 시 `status.md` 우선”이 있다
6. 파일 부재 시 기존 목록 조회로 폴백하는 경로가 문서에 있다
7. 세션 훅(동작 호스트): 파일 없으면 생성, 있으면 유지, 실패해도 세션 계속
8. 장부 없는 cwd·개발 모노레포에서는 훅이 현황 파일을 만들지 않는다
