## 범위

기준일 `asOf`에서 **부채(liability) 계정 잔액**을 유동·비유동으로 나눈다.
읽기 전용이다. 스키마·전표·계정 코드를 바꾸지 않는다.

자산 구분, 이자·수수료 분류, 계정 트리 재편은 범위 밖이다.

## 사람 말로

보고일 기준으로 “1년 안에 나갈 원금”과 “그 뒤 원금”을 본다.
카드값·미지급은 늘 단기다. 할부·대출은 이미 있는 상환·할부 예측의 **원금 줄**만
본다. 예측과 장부 잔액이 어긋나면 장부 잔액에 맞추고, 남는 차이는 유동에 붙인다.

## 인터페이스

| 표면 | 동작 |
|---|---|
| 잔액 조회 (`balance`) | 결과에 만기 구분 요약(및 계정별 행)을 포함한다. `--as-of`가 기준일이다. 생략 시 오늘 |
| 마감 (`close` / `--dry-run`) | 마감 기준일(기간 말일)으로 같은 요약을 결과에 붙인다 |
| 스킬·대화 | “유동부채” 질의 시 위 조회와 같은 숫자를 읽는다 |

새 변경 명령은 없다. JSON 키·플래그 세부는 CLI 스펙 페이지에서 맞춘다.

## 동작

입력: 기준일, 부채 계정별 잔액(기능통화 weight), 할부·대출 스케줄(있을 때).

창: `(asOf, asOf + 12 calendar months]` — 기준일 **다음날**부터 12개월이 끝나는
날까지 만기가 오는 원금을 유동으로 본다.

계정별:

| 계정 | 유동 | 비유동 |
|---|---|---|
| 카드 부채(할부 계정 제외) | \|잔액\| 전액 | 0 |
| 미지급 등 그 외 부채(스케줄 없음) | \|잔액\| 전액 | 0 |
| 할부 부채 | 창 안 회차 **원금** 합 | 창 밖 회차 원금 합 |
| 대출 부채 | 창 안 스케줄 **원금** 합 | 창 밖 스케줄 원금 합 |

부호: 장부에서 부채 잔액은 보통 음수(대변)다. 구분 금액은 **빚의 크기(절댓값)**
로 보고하고, JSON에서는 기존 weight 부호 규약과 맞춰 문서화한다.
어느 쪽이든 **유동+비유동 = 그 계정 \|잔액\|** 을 깨면 안 된다.

### 스케줄과 잔액이 다를 때

1. 스케줄로 나눈 유동·비유동 원금 합을 `S`라 하고, 장부 \|잔액\|을 `B`라 한다
2. `S == B`이면 그대로 쓴다
3. `S < B`이면 부족분 `B - S`를 **유동에 더한다**
4. `S > B`이면 유동·비유동을 `B/S` 비율로 줄여 합이 `B`가 되게 한다.
   최소 화폐 단위 정수는 유동 쪽을 마지막으로 맞춰 합이 정확히 `B`가 되게 한다

이자는 스케줄에 있어도 구분 합에 넣지 않는다.

## 데이터와 불변식

| 사람이 읽는 이름 | 코드 키(안내용) | 예시 |
|---|---|---|
| 기준일 | `asOf` | `2026-07-31` |
| 유동부채 합계 | `currentMinor` | `4200000` |
| 비유동부채 합계 | `nonCurrentMinor` | `86000000` |
| 부채 총액 | `totalMinor` | `90200000` |
| 계정별 유동/비유동 | `byAccount[]` | 주담대 행 |

불변식:

- `currentMinor + nonCurrentMinor === totalMinor`
- `totalMinor ===` 포함 부채 계정의 \|weight\| 합(같은 기능통화)
- 전표·스냅샷 스키마 변경 없음 (1차)

## 오류

이 계약은 조회 결과만 만든다. 새 거부 코드는 없다.
원장 열기·잔액 조회 실패는 기존 `balance` / `close` 오류를 따른다.

## 호환성

- 마이그레이션 없음
- 계정 코드·루트 불변
- 기존 `balance` / `close` 사용자는 필드가 **추가**될 수 있다. 제거·의미 변경 없음

## 수락 기준

1. 카드만 있는 fixture → 전액 유동, 비유동 0
2. 남은 원금 회차가 18개월인 할부 → 유동 12개월분 + 비유동 6개월분, 합 = 잔액
3. 대출 스케줄 원금 합 ≠ 장부 잔액 → 합은 장부 잔액, 부족분은 유동
4. `close --dry-run`과 `balance --as-of <말일>`의 합계가 같다
5. 구분 호출 전후 `txn` 행 수 불변
