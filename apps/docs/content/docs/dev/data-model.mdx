버전은 이 스펙(스키마 마이그레이션)의 번호다. CLI 0.1.0·도메인 정책과 동기화되지
않는다.

`__holiday_migrations`가 적용분마다 sha256을 남긴다. 적용된 파일을 수정하면 원장
열기가 거부된다 — [POLICY-016](/docs/domain/policy#policy-016).

## Postgres 마이그레이션

`store-postgres`는 SQLite의 5단계를 **스쿼시**한다.

| Postgres 폴더 | 내용 |
|---|---|
| `20260717115119_init` | 현재 테이블 일식 생성 (loan·ingest·close 포함) |
| `20260717115212_invariant_triggers` | plpgsql 트리거 11개 |

기존 SQLite 원장을 Postgres로 이어 받는 경로가 아니라, 엔진별 스키마 생성물이다.

맞추지 않는다.

## fx_rate · period · close

- `fx_rate`: `(as_of, base, quote, source)` 유니크. 기표된 weight를 바꾸지 않는다.
  이후 유도·재평가에만 사용.
- `period`: grain·구간·`open|closed|locked`.
- `balance_assertion`: 명세서 잔고. 외부와 대조하는 유일한 저장값.
- `snapshot` / `snapshot_balance`: 마감 시점 계정별 units/weight와 기간 증분.

## 재평가 전기 형태

마감(`holiday close`)은 화폐성 자산·부채의 외화 잔고에 대해 weight만 조정한다.
`units_minor = 0`, `weight_minor ≠ 0`이 가능하려면 weight가 units×rate에서
유도되지 않고 **저장**되어 있어야 한다.

상대 계정은 `Income:FX:Unrealized`다. `system_kind = fx_revaluation`.

<LedgerExample
  date="2026-07-31"
  narration="월말 재평가"
  legs={[
    { account: 'Assets:Bank:Wise:USD', units: '0 USD', weight: 100000, source: 'actual', note: '외화 units 불변, KRW carrying만 이동' },
    { account: 'Income:FX:Unrealized', units: '-100,000 KRW', weight: -100000, source: 'identity' },
  ]}
/>

재평가는 기간이 아직 `open`인 동안 기표한 뒤 스냅샷을 쓰고 `closed`로 전환한다.
비용·수익 계정은 재평가하지 않는다 (IAS 21 화폐성 자산·부채).
