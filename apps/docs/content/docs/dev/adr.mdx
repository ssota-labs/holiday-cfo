---
title: ADR
description: 채택·거부 결정 기록
type: decision
summary: 거부한 대안이 본문이다. status는 가역성을 나타낸다.
---

각 Decision의 본문은 **거부한 대안**과 채택 근거다. 채택된 설계는 코드에도 남지만,
시도 후 버린 경로는 코드에 남지 않는다. `status`:

| 값 | 의미 |
|---|---|
| `accepted` | 현행. 변경 가능 |
| `locked` | 과거 데이터 재작성 없이는 변경 불가 |

## ADR-001 … ADR-006

<Decision
  id="ADR-001"
  title="환율이 아니라 상대금액으로 균형을 잡는다"
  status="locked"
  date="2026-07-17"
  commit="836f6c5"
  rejected={[
    { option: '거래마다 기표통화 + 전기마다 환율', because: '환율을 되곱하면 정수 스케일에서 lossy하다. ₩1,000,000 → $750.00은 1333.3333…이고 역산하면 ₩999,998. 허용오차를 두면 잡고자 하는 크기(누락 수수료 등)를 가린다.' },
    { option: 'Beancount의 통화별 weight 그룹', because: '기준통화 없이 균형을 잡을 수 있는 대신 허용오차 추론과 booking 알고리즘이 필요하고, KRW 장부가액이 명시되지 않아 매번 lossy 변환이 필요하다.' },
  ]}
>
전기마다 `units_minor`(사실)와 `weight_minor`(측정)를 둔다. weight는 Beancount
`@@`(총액)에 해당하는 값을 환율에서 곱해 내지 않고 직접 기록한다.

**locked:** 기존 전기가 “이 사건의 기능통화 측정”을 이미 인코딩한다. 기능통화를
바꾸려면 과거를 `units × rate`로 재유도해야 하며, 이는 본 설계가 피하는 연산이다.
새 국면은 마이그레이션이 아니라 새 book으로 연다.
</Decision>

검증한다. **제품 CLI 경로는 SQLite만** 연다 (`workspace.ts`). 원격 Postgres를

런타임에 읽지 않고 인라인된 마이그레이션을 쓴다.

대가: STRICT 테이블 미지원(drizzle-kit). RC 핀(`1.0.0-rc.4`). stable에는
node:sqlite 드라이버가 없다.
</Decision>

<Decision
  id="ADR-006"
  title="스토어는 하나. 방언만 갈라진다"
  status="accepted"
  date="2026-07-17"
  commit="2c90bef"
  rejected={[
    { option: 'store-sqlite를 기계 변환해 store-postgres 복제', because: '초기에는 일치해도 첫 손수정에서 갈라진다. 포트 경계를 두는 목적과 반대다.' },
    { option: 'Drizzle로 스키마 하나 공유', because: 'ADR-005. 빌더·트리거 본문이 엔진별로 다르다. 스키마 비공유 ≠ 스토어 이중 구현.' },
    { option: 'node:sqlite에 맞춰 스토어를 동기로 유지', because: 'Postgres는 동기화할 수 없다. 동기→비동기 래핑은 싸고, 반대는 불가능하다.' },
  ]}
>
측정: 스토어 약 1,486줄 중 SQLite 고유는 `GLOB`·`PRAGMA`·`INSERT OR IGNORE`·
`BEGIN IMMEDIATE` 합 8줄. 구조적 차이는 드라이버 동기성뿐이다.

`@holiday/store-sql`이 `LedgerStore`를 하나 구현한다. `store-sqlite` /
`store-postgres`는 드라이버·스키마·마이그레이션이다. Dialect:
`subtreeWildcard()`, `prepare()`, 선택적 `init`/`checkpoint`.

공유 SQL의 `GLOB`은 대소문자 구분 매칭을 위해 남긴다. Postgres `LIKE`는 이미
구분한다. 드라이버가 방언을 옮긴다.

판단 근거: 적합성 18케이스가 리팩터 전후 동일 통과. Postgres 어댑터도 동일
스위트로 검증한다 (pglite).
</Decision>

## ADR로 아직 고정하지 않은 관행

문서화만 하고 Decision id를 두지 않은 항목:

| 관행 | 현재 동작 |
|---|---|
| Draft ingest | `ingest submit` → draft. 잔액 제외. `review accept` 후에야 posted |
| Close without closing entries | `close`는 FX 재평가·스냅샷·기간 `closed`까지 수행. 손익 마감분개(`closing_entry`)는 스키마 enum에만 있고 CLI 경로는 없음 |

필요 시 후속 ADR로 승격한다.
