## SKILL.md 규칙 요약

1. 금액·날짜·계정을 지어내지 않는다. 불명확하면 질문한다.
2. 거래는 정확히 균형. `unbalanced`를 숫자 보정으로 “고치지” 않는다.
3. 계정은 `account list` 결과를 쓴다. 없으면 생성안을 제안한다.
4. 비자명 기표는 전표를 말로 보여 확인받은 뒤 쓴다.

출력 계약: `--json` → stdout. 에러 → stderr 단일 JSON 봉투.

## 캡쳐 → ingest → review

OCR은 CLI가 하지 않는다. 모델이 파싱한 legs를 `ingest submit`한다. 결과는
**draft**이며 잔액·리포트에서 제외된다. 사람이 `review accept`해야 posted다.

```bash
holiday ingest submit --idem-key K1 --data '{ ... }'
holiday review list
holiday review accept <id>
holiday review reject <id> --reason "..."
```

`--idem-key`로 타임아웃 재시도 시 재생한다. `externalRef`가 있으면 중복은 거부로
처리한다. draft도 균형이어야 한다.

게이트는 잘못된 카테고리를 걸러낸다. 오독한 금액은 사람이 승인하면 통과한다.
금액이 불명확하면 멈춰 질문한다.

## 아직 못 하는 것

SKILL.md 말미와 동일하다. 즉흥으로 메우지 않는다.

| 항목 | 상태 |
|---|---|
| OCR | 없음. 모델이 파서. 이미지는 해시용 |
| 자동 승인 | 없음. draft마다 사람 확인 |
| 할부수수료 공식 | 계산하지 않음. 명세서 `--fees` |
| 환율 API 자동 조회 | 없음. `fx add`로 수동 입력. 없으면 throw |
| 대시보드 / 원격 SoR UI | 없음. 제품 CLI는 로컬 SQLite |

ingest·review·fx·assert·close는 **존재한다.** 위 표에 넣지 않는다.
