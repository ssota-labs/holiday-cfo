제품 진입점은 CLI다. 스킬·에이전트는 이 명령을 shell-out한다. 구현 기준은
`packages/cli/src/cli.ts`다.

## 전역 계약

| 항목 | 규약 |
|---|---|
| `--json` | stdout에 기계 가독 JSON. 사람용 안내는 stderr로만 출력하며 `--json`이면 생략 |
| 에러 봉투 | 실패 시 stderr에 단일 JSON: `{"error":{"code":"...","message":"..."}}` |
| exit 0 | 성공 |
| exit 1 | 원장/도메인 거부 (`LedgerError`: `unbalanced`, `assertion_failed`, `close_blocked` 등) |
| exit 2 | 사용법 오류 (`UsageError`: 없는 계정, 파싱 실패, 워크스페이스 없음) |

```json
{"error":{"code":"unbalanced","message":"postings do not balance: they sum to 50 KRW minor units…"}}
```

금액은 JSON 경계에서 **소수 문자열**로만 교차한다. `number`/`bigint` JSON 직렬화를
쓰지 않는다.

`--data` 등 명령 전용 JSON 입력 플래그 이름은 전역 `--json`과 겹치지 않는다.
commander가 한쪽만 해석하고 나머지를 무시하기 때문이다.

## 명령 목록

`mutates`는 원장 또는 워크스페이스 상태를 쓰는지 여부다.

| 명령 | mutates | 요약 |
|---|---|---|
| `init` | yes | `.holiday/` 생성, 기능통화·close grain·timezone, commodity seed |
| `account add <code>` | yes | 계정 추가 |
| `account list` | no | 계정 목록 |
| `txn add` | yes | 거래 기록 (`--draft` 가능) |
| `balance` | no | 잔액 (`--as-of`, `--account`) |
| `card add <code>` | yes | 카드 청구주기 부착 |
| `card list` | no | 카드·청구규칙 |
| `installment add` | yes | 할부 구매 + 스케줄 |
| `installment revise <id>` | yes | 스케줄을 명세서 값으로 덮어씀 |
| `installment list` | no | 잔여 할부 |
| `recurring add <label>` | yes | 정기지출 등록 |
| `recurring list` | no | 활성 정기지출 |
| `loan add <code>` | yes | 대출 상환 스케줄 부착 |
| `loan list` | no | 대출 현황 |
| `loan check [code]` | no | 스케줄 vs 원장 대사 |
| `loan pay <code>` | yes | 스케줄 기준 원리금 분할 기표 |
| `fx add <base> <quote> <rate>` | yes | 환율 기록 (기표된 weight는 불변) |
| `fx list` | no | 환율 목록 |
| `fx show <base> <quote>` | no | 해석될 환율과 근거 |
| `assert <account> <amount>` | yes | 잔액 대사 기록·대조 (실패 시 exit 1) |
| `close <month>` | yes | 월 마감 (재평가·스냅샷·`period=closed`). `--dry-run`은 쓰기 없음 |
| `ingest submit` | yes | 파싱된 거래를 draft로 적재 |
| `review list` | no | pending draft |
| `review accept <id>` | yes | draft → posted |
| `review reject <id>` | yes | draft 거부 (행은 유지) |
| `cashflow` | no | 현금 런웨이 투영 |
| `verify` | no | 원장·감사 체인 검사 |
| `checkpoint` | yes | WAL을 `ledger.db`로 checkpoint |

스케줄·데이터 수집·마감의 도메인 함정은 [Features](/docs/features/cashflow)와 각 feature
페이지에 정리한다.

## 핵심 명령 상세

<CommandSpec
  command="txn add"
  mutates={true}
  summary="거래를 기록한다. 다리들의 weight 합이 정확히 0이 아니면 거부한다."
  signature={`holiday txn add
  --leg "ACCOUNT AMOUNT COMMODITY [@@ TOTAL]"
  [--leg ...]
  [--date YYYY-MM-DD] [--payee NAME] [--narration TEXT] [--draft]`}
  flags={[
    { name: '--leg', type: 'string', required: true, repeatable: true, description: 'ACCOUNT AMOUNT COMMODITY. 기능통화가 아니면 "@@ TOTAL" 또는 환율 테이블 유도' },
    { name: '--date', type: 'YYYY-MM-DD', default: 'today' },
    { name: '--payee', type: 'string' },
    { name: '--narration', type: 'string', default: '""' },
    { name: '--draft', type: 'boolean', default: 'false', description: '검토 대기. 잔액·리포트에서 제외' },
  ]}
  exits={[
    { code: 0, meaning: '기록됨. stdout에 {id, status, fxEstimated}' },
    { code: 1, meaning: '원장 거부 — unbalanced 등' },
    { code: 2, meaning: '사용법 오류 — 없는 계정, 파싱 실패' },
  ]}
  example={`$ holiday txn add --date 2026-07-17 --payee "이마트" \\
    --leg "Expenses:Food:Groceries 42000 KRW" \\
    --leg "Liabilities:Card:Shinhan -42000 KRW"
{"id":"01KXQ…","status":"posted","fxEstimated":false}`}
/>

`@@`는 **총액(TOTAL)**이다. 단위당 환율 `@`는 지원하지 않는다. 환율을 되곱하면
정수 스케일에서 손실이 나고, 허용오차를 강제하게 되기 때문이다.

<CommandSpec
  command="cashflow"
  mutates={false}
  summary="오늘 현금에서 카드 청구·할부·정기지출·대출 상환을 빼며 투영한다."
  signature={`holiday cashflow [--until YYYY-MM-DD] [--json]`}
  flags={[
    { name: '--until', type: 'YYYY-MM-DD', default: '3개월 후', description: '투영 지평' },
  ]}
  exits={[
    { code: 0, meaning: '정상. 잔고가 음수여도 0 — SHORT는 결과이지 에러가 아님' },
    { code: 2, meaning: '원장 없음' },
  ]}
  example={`$ holiday cashflow --until 2026-10-31
cash on hand (2026-07-17):  3000000 KRW

2026-10-01   -      117000   →        -17000   ⚠ SHORT
             냉장고 (2/12)                           100000
             넷플릭스 (2026-08-17 결제분)                 17000

⚠ Short by 17000 KRW on 2026-10-01.`}
/>

카드 funding 정기지출은 발생일이 아니라 청구주기를 탄다. 출력에 결제 기준일을
함께 표시한다.

`close`는 기간 내 draft 미해소 또는 실패한 잔액 대사가 있으면 거부한다
(`close_blocked`). 통과 시 FX 재평가 → 스냅샷 → `period.status=closed`다.
닫힌 기간으로의 추가 기표 거부는 아직 store에 없다. 상세는
[잔액 대사와 월 마감](/docs/features/close)·[데이터 모델](/docs/dev/data-model)을 본다.

  command="checkpoint"
  mutates={true}
  summary="WAL을 ledger.db로 접는다. git 커밋 전에 실행한다."
  signature={`holiday checkpoint`}
  exits={[{ code: 0, meaning: 'ledger.db 커밋 가능' }]}

`ledger.db`는 커밋 대상이다. `-wal`/`-shm`은 `.gitignore`된다. checkpoint 없이
커밋하면 최근 변경이 WAL에만 남을 수 있다.
