`agent-bowl`은 서로 독립적으로 관리되는 에이전트 생태계 패키지를 찾고, 조합해, 설치 가능한 App으로 만드는 개인용 플랫폼이자 레지스트리 구상이다. 패키지 원문을 이 저장소에 복제하지 않고 각 upstream의 출처와 갱신 주기를 유지한다. App은 패키지 내용을 소유하는 대신 manifest와 정확한 lock으로 구성을 정의한다.

현재 단계는 토론 및 설계 초안이다. 스키마, 명령 이름, 지원 범위는 확정되지 않았다. 이 저장소는 투자 분석 App이나 그 구성 패키지를 구현하지 않는다.

## 원칙

- Package와 App을 구분한다. Package는 독립된 출처와 버전을 가진 재사용 단위이고, App은 여러 Package의 조합이다.
- 원본보다 색인을 우선한다. 레지스트리는 canonical source를 가리키며 패키지 내용을 임의로 vendoring하지 않는다.
- 원하는 상태와 실제 상태를 분리한다. manifest, lock, provenance, machine-local receipt가 서로 다른 사실을 기록한다.
- 설치 전에 의존성, 출처, 권한, 라이선스를 확인할 수 있어야 한다.
- Cursor, Claude Code, Codex에는 각 호스트가 이해하는 네이티브 결과물을 만들고, 지원하지 못하는 기능은 숨기지 않는다.
- 로컬과 향후 cloud 실행은 같은 App manifest와 lock을 사용한다.

## 하지 않는 일

- 모든 생태계 프로젝트를 하나의 저장소나 배포 주기로 합치지 않는다.
- 표시 이름을 전역 식별자로 간주하지 않는다.
- symlink 구조를 설치 계약으로 삼지 않는다.
- 수정된 사용자 코드를 scaffold가 덮어쓰게 하지 않는다.
- 임의 코드를 담은 보고서 형식을 신뢰 경계 안으로 들이지 않는다.
- 현재 설계 초안을 확정된 스키마나 CLI 계약으로 제시하지 않는다.
